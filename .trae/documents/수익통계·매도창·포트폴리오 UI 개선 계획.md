## 목표
- 수익통계 기본 기간을 최근 1년·월별 집계로 초기 표시하고, 명시적 ‘검색’ 버튼과 자산 유형(카테고리) 필터 추가
- 매도창 ‘보유정보’ 시각적 구분 개선
- 포트폴리오 테이블의 일부 컬럼 기본 숨김 및 아코디언 확장/축소 제공, 반응형 그리드 최적화
- 매도 아이콘을 업계 표준 스타일로 개선(가시성/직관성/툴팁 최적화)

## 변경 대상 파일
- `components/SellAnalyticsPage.tsx` 수익통계 UI/로직 개선
- `components/SellAssetModal.tsx` 매도창 헤더/섹션 스타일 개선
- `components/PortfolioTable.tsx` 테이블 컬럼 표시 제어/아코디언 추가/반응형 조정
- `App.tsx` 탭·상태 연결(필요 시 경량 상태 추가)
- `index.html` (옵션) Material Symbols 링크 추가 또는 인라인 SVG 아이콘 교체

## 수익통계 개선 상세
- 초기 기간/집계 설정
  - 컴포넌트 마운트 시 `startDate = (오늘 - 365일)`, `endDate = 오늘`, `grouping = 'monthly'` 기본값 설정
  - 현재 즉시 필터 반영 방식에서 ‘입력 상태’와 ‘적용 상태’를 분리하여 ‘검색’ 버튼 클릭 시만 적용
- 명시적 ‘검색’ 버튼
  - 상단 필터 패널에 버튼 추가: 클릭 시 입력된 기간/카테고리/검색어/집계를 ‘적용 상태’로 커밋
  - 적용 시 차트/표/히스토그램을 모두 재계산(useMemo 의존성에 적용 상태 사용)
- 자산 유형 필터
  - `AssetCategory` 기반 셀렉트 추가(예: 전체/한국주식/미국주식/채권/암호화폐/실물자산 등)
  - 필터 로직: `SellRecord.category` 기준으로 거래 집합을 제한
- 사용자 경험
  - 월별 라인차트 기본 표시, 툴팁/범례/축 포맷팅 유지
  - CSV 내보내기 시 현재 ‘적용 상태’ 기준으로 생성

## 매도창 UI/UX 개선
- ‘보유정보’ 박스 헤더를 입력 필드 라벨과 동일한 위계로 표시(굵기/간격/경계선)
- 박스 내부 레이아웃 정렬·여백 개선(Tailwind 클래스 통일), 시각적 일관성 향상

## 포트폴리오 레이아웃 최적화
- 기본 숨김 컬럼: `보유수량`, `매수일`, `매수평균가`, `비중`
  - 상단에 ‘세부 항목 보기’ 토글을 추가(전역), 끄면 위 4개 컬럼을 숨김
- 아코디언 확장/축소(행 단위)
  - 각 행의 좌측/우측에 확장 버튼 추가, 숨김 항목을 행 아래 서브패널로 표시(반응형 그리드로 2열 배치)
  - 기존 개별 손익 차트 확장(이미 구현)과 충돌 없도록 별도 패널로 구성
- 반응형 그리드 조정
  - `sm`/`lg` 브레이크포인트에서 컬럼 폭/줄바꿈 최적화, 모바일에서 스크롤 과도화 방지

## 매도 아이콘 디자인 개선
- 적용 방식 A(권장): `index.html`에 Google Material Symbols(Outlined) 링크 추가 후 `span.material-symbols-outlined`로 아이콘 사용
  - 장점: 경량, 접근성 개선(텍스트 대체), 일관된 디자인
- 적용 방식 B(라이브러리 미도입): 인라인 SVG를 Material 가이드라인과 유사한 ‘sell’/‘shopping_cart_checkout’ 형태로 교체
- 상호작용 개선
  - 색상: 경고/행동 유도 컬러(`text-red-400` → 가시성 높은 톤 유지), hover 상태 명확화
  - 크기: `h-5 w-5` 기본, 모바일에서 터치 타겟 확대
  - 툴팁: 의미 명확한 설명 문구로 업데이트(예: “선택한 자산 매도”)

## 정확성·성능
- 정확성: 필터 적용 상태 분리로 사용자의 의도와 데이터 일치, 월별 집계는 날짜 키를 `YYYY-MM`로 그룹화
- 성능: 메모이제이션으로 재계산 범위 최소화, 대규모 데이터에서도 3초 응답 목표 유지

## 테스트/검증
- 단위 테스트(로직): 월별 그룹화/기간 필터/카테고리 필터/CSV 생성
- 시각 테스트(UI): 모바일/데스크톱 반응형 확인, 아이콘/툴팁 접근성 점검
- 통합 검증: 빌드/프리뷰로 탭 간 전환, ‘검색’ 클릭 후 데이터 반영 확인

## 구현 순서
1) `SellAnalyticsPage.tsx`에 초기값/검색 버튼/카테고리 필터/적용 상태 추가, 월별 기본 표시
2) `SellAssetModal.tsx`의 ‘보유정보’ 섹션 헤더/레이아웃 개선
3) `PortfolioTable.tsx`에 전역 ‘세부 항목 보기’ 토글과 행 아코디언 추가, 반응형 조정
4) 매도 아이콘을 표준 스타일로 교체 및 상호작용 개선
5) 빌드/프리뷰로 통합 검증, 미세 조정

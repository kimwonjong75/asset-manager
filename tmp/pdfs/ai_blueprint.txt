3.2 작업 철학 — Claude Code 운영 8대 원칙 (절대 규칙)
모든 작업 세션에 자동 적용되는 8가지 원칙. 이것이 구축의 방법론 그 자체다.
1. 코드베이스 Q&A 우선 — 코드를 짜기 전에 먼저 탐색해 이해한다.
2. git 히스토리로 "왜"를 이해 — 무엇이 바뀌었나가 아니라 왜 바뀌었나를 본다.
3. 프로젝트 규칙 자동 로드 — 회사 규칙·금지사항을 매 세션 자동으로 읽는다.
4. 실행 전 계획 → 승인 — 다파일 변경/DB변경/외부발송 전 반드시 계획부터.
5. 테스트·스크린샷 자기검증 — 결과를 스스로 검증한다.
6. AI 기억 직접 점검 — AI의 기억을 직접 확인·정리한다.
7. MCP/SDK로 직접 연결 — 로그·git·슬랙·노션을 도구로 직접 붙인다.
8. worktree/병렬 실행 — 독립 작업은 격리된 작업공간에서 병렬로.
3.3 절대 안전 규칙 (위반 시 시스템이 차단)
1. 발주 자동 전송 금지 — 발주서는 초안만. 반드시 경영진 컨펌 후.
2. 외부 발송 승인 필수 — 슬랙·이메일·발주 모두 초안 먼저 확인.
3. 기술 막힘 시 수동 요청 — 자동화 안 되면 재시도 말고 즉시 사람에게.
4. 시간대 표준 고정 — 운영 기준 시간대를 명시하고, 저장은 UTC·표시는 변환으로 통일.
5. 삭제·force·검증우회 금지 — 훅으로 자동 차단.
이 규칙들은 단순 권고가 아니라 코드(가드)와 훅에 물리적으로 박혀 있어 AI가 어겨도 막힌다.

3.4 작업 메모리 — 세션을 넘어 기억하기
AI는 대화가 끝나면 기억을 잃는다. 이를 막기 위해 파일 기반 영속 메모리를 운영한다.
사실 1개당 파일 1개 + 색인 파일 구조. 매 세션 시작 시 자동 로드되어 "지난번에 무엇을 어떻게 고쳤는지"를
이어받는다.
수십~수백 개의 메모리 파일이 누적되어 자동화 의사결정·사고복구 이력이 보존된다.

7. 자율진화 — 스스로 코드를 고치는 엔진
7.1 개념 — "무인 자율 코딩"
auto-evolve 는 시스템이 스스로 자기 코드의 문제를 찾아 고쳐 커밋하는 엔진이다. 사람이 자는 동안에도 작은 개선이

7.2 4단계 사이클 (하루 10회)
① Discover (탐색, 비용 $0)
6종 신호 스캔 → 할 일 목록 작성
1) 코드 TODO/FIXME 2) 스케줄러 에러로그 3) 린트 위반
4) 제안 큐 5) 스킬 개선 신호 6) 자율진화 메타 신호
② Synergy (연계, 비용 $0)
프로젝트 간 중복·연결 가능성 탐지
③ Runner (실행, task당 소액)
최고점수 task 1건 선택
→ [가드 A: 진입 검증] → git worktree 격리 → Claude 패치 작성

9. 데이터는 어디서 와서 어떻게 표현되나
한 상품의 데이터를 예로 들어 수집 → 저장 → 표현의 전 과정을 추적해 보자.
[수집] 이커머스 플랫폼에 신규 상품 등록
└→ 동기화 스크립트(1시간 주기)가 재고·상품을 긁어옴
[저장] 클라우드 PostgreSQL 마스터 DB의 상품 테이블에 업서트
└→ ORM (발주 ERP 앱)
[가공] 가격 백필·마진 계산·이미지 동기 (자동화 스크립트)
[표현] 통합 허브가 읽기전용 쿼리로 SELECT
└→ 화면에 카드/테이블로 렌더
[보고] 적자 상품·재고 임계 → 매일 아침 책임자 슬랙 DM 브리핑
핵심 원칙:
마스터 데이터는 한 곳에. 다른 앱은 읽기 전용으로 본다 → 데이터 꼬임 방지.
수집은 자동, 표현은 실시간 쿼리. 화면은 DB를 직접 읽으므로 항상 최신.
슬랙·드라이브 같은 비정형 데이터는 크롤러가 긁어 파서로 구조화 후 DB/JSON에 적재.

11.2 자율진화 무산출 (여러 차례 재발 → 매번 다른 근본원인)
차수 근본 원인 해결
1차 테스트 베이스라인 red (다수 원인) 전량 green 복구
2차 pool 데드락 + 미커밋 + worktree 환경노이즈 delta validate 도입
3차 runner가 worktree symlink를 커밋해 의존성 파손 upstream 복원 + 경로 제외
4차 git 버전 차이로 인한 비정상 종료 명령 단순화로 수정
5차 무의미 산출 퇴화 (노이즈 연료, 실코드 시야밖) 노이즈 차단 + 코드 스캔 확대 + 빈 사이클 종료
교훈: "working tree green ≠ HEAD green". 자율 시스템은 무산출도 비용이므로 빈 사이클은 종료시켜야 한다.

참고용이며, 실제 절감액은 가동 범위에 따라 달라진다.
투입 비용 — 시스템 운영비 약 월 수십만원 규모.
같은 역할을 외부에서 사면 드는 돈 (시장가 개략):
시스템이 대체·보조하는 일 시장가격(월, 개략)

통관·VAT·법률 1차 검토 (건당 외주) 수십만원/건
개발·유지보수 (자율진화가 일부 대체) 변동

대체가치 월 수백만원 규모, 투입 대비 약 10배 이상 효율(개략).
측정 가능한 직접 효과(유형):

AI 운영비 폭증 차단 — 모델 강등으로 비용 대폭 절감.
24시간 자동 크롤·브리핑 — 사람이 매일 수동 점검할 시간(일 1~2시간/인)을 회수.

13. 도입 로드맵 — 다른 조직에 어떻게 이식하나
13.1 복제 가능성 — 왜 그대로 이식되나
이 시스템은 회사 특화 데이터를 제외하면 구조가 범용적이다. 스케줄러·MCP·AI 팀·자율진화·승인게이트는 어떤
조직에도 동일하게 적용된다. 각 조직이 자기 데이터(자기 이커머스·슬랙·DB)에 연결만 하면 같은 체계를 복제할 수 있다.
13.2 단계별 도입 로드맵
Phase 0 — 토대 (1~2주)
· Claude Code + Full 프로필 설치
· 회사 규칙·금지사항·호칭·시간대를 규칙 파일로 작성
· MCP 연결: 자기 슬랙·메일·드라이브·노션·DB
Phase 1 — 수집·표현 (2~4주)
· 크롤러 배선 (슬랙·드라이브·이커머스 등)
· 마스터 DB 1곳 + 읽기전용 대시보드 1개 (통합 진입점 URL)
· 매일 1회 책임자 브리핑
Phase 2 — 실행 (1~2개월)
· 분야별 AI 팀 페르소나 정의 (조직도 본떠)
· 디스패처 + 승인 게이트
· 스케줄러 등록
Phase 3 — 진화·연계 (지속)
· 자율진화 엔진 (안전모드부터)
· 조직 간 데이터 브리지 (시너지)
13.3 개발자 투입 시 우선 고도화 영역
하드코딩 개발자를 붙일 경우 ROI가 높은 순서:
1. 배송추적·이메일마케팅 실연동 (배송추적·이메일 자동화 완성)
2. 소셜·이커머스 웹훅 실시간화 (배치 → 실시간)
3. 헤드리스 OAuth 세션 안정화 (종량 과금 근본 차단)
4. 조직 간 데이터 브리지 (그룹 시너지의 핵심)
5. AI 팀 산출 → 실행 자동화 범위 확대 (승인게이트 유지 하에)

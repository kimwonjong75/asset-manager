## 0. 대원칙 — 이것 하나만 기억하면 된다

> **실제로 사고파는 근거는 실행 큐 하나다. 나머지는 전부 참고와 연구다.**

이 앱의 다섯 기능은 권한이 서로 다르다. 권한을 섞으면 (예: 참고 지표를 보고 매도하면) 시스템이 무너진다.

| 기능 | 위치 | 권한 | 한 줄 요약 |
|---|---|---|---|
| **실행 큐** | 실행 큐 탭 | 🔴 **실행** — 여기 뜬 주문만 실제로 낸다 | 오늘의 주문서 |
| **구루 신호 엔진** | 대시보드 상단 카드 | 🟡 참고 — 지켜볼 후보를 알려줄 뿐 | 오늘의 관찰 후보 |
| **참고 지표 (리스크 매트릭스)** | 대시보드, 구루 카드 아래 접힘 | 🟡 참고 — 과열 경고등, 행동 지시 아님 | 과열 경고등 |
| **③ 대표 매수 종목 지정** | 리밸런싱 화면 | ⚙️ 설정 — 반기 주문 생성의 전제 조건 | 부족하면 뭘 살지 예약 |
| **신호 리플레이** | 신호 리플레이 탭 | 🔵 연구 — 라이브 신호에 영향 없음 | 과거 복기 훈련장 |

루틴 전체 지도:

```text
매일 5분   실행 큐 — 주문 확인·실행
매일 3분   대시보드 — 구루 신호 훑기 + 과열 배지
주 1회 30분 신호 리플레이 — 복기·판정 기록
월 1회 10분 정비 — 터틀 후보·위성 예산·놓친 신호
반기 1회 1시간 코어 리밸런싱 — 목표비중·③ 대표 매수 종목·주문 생성
```

## 1. 매일 5분 — 실행 큐 (유일한 매매 시간)

장 마감 후(또는 하루 중 정한 시간에) 한 번.

- [ ] **1-1. 실행 큐 탭에 들어가 [오늘 주문 생성] 버튼을 누른다.**
  - 화면에 들어가기만 해서는 갱신되지 않는다. 버튼을 눌러야 오늘 기준으로 다시 계산한다.
- [ ] **1-2. 주문이 있으면** — 카드를 읽고 실제 계좌에서 체결한 뒤 **[실행 완료]**(또는 [실행하기])를 누른다.
  - 터틀 주문(신규 매수·불타기 추가·손절 매도·청산 매도)은 전용 모달이 열린다. 체결 가격·수량을 입력하고 저장.
  - 실행하지 않기로 했으면 **건너뜀 + 사유**를 남긴다. 사유 없이 방치 금지 — 3일 이상 방치하면 에스컬레이션 표시가 뜬다.
- [ ] **1-3. 주문이 0건이면** — **"왜 주문이 없나요?"** 패널을 펼쳐 한 번 훑는다.
  - 정상 문구(조치 불필요): `55일 신고가 미돌파(정상 대기)` · `이미 대기 중인 주문이 있어…` · `보유 포지션 …조건 미충족(정상 대기)`
  - ⚠️ `가격 데이터(OHLCV)·N 부족으로 판정 불가: [종목명]` 에 **위성 9종목 중 하나라도 뜨면** 계기판 고장이다. 매매 판단이 아니라 코드 문제이니 Claude에게 알린다.

주문이 0건인 날이 대부분이다. **0건 = 정상**. 시스템이 "오늘은 할 일 없음"이라고 말하는 것이다.

## 2. 매일 3분 — 대시보드 훑기 (읽기만 한다)

여기서는 아무것도 사고팔지 않는다. 눈으로만 본다.

- [ ] **2-1. 구루 신호 엔진 카드** — 매도 경고부터 본다.
  - `매도 경고`에 보유 종목이 있으면: **[왜 떴나]**를 눌러 조건을 읽는다. 바로 팔지 않는다 — 실제 매도는 실행 큐의 손절/청산 주문, 또는 반기 리밸런싱이 담당한다.
  - `관찰 후보` / `진입 검토`에 뜬 종목: 궁금하면 [왜 떴나]로 근거만 확인. **신호 ≠ 매수 지시**다. 카드에도 적혀 있다: "지식 규칙 기반 관찰 후보 · 매수 추천이 아닙니다".
  - 신호가 하나도 없으면: 정상. 궁금하면 **[왜 신호가 안 뜨나요?]** 진단으로 조건 미충족/데이터 부족/미지원을 구분해 본다.
- [ ] **2-2. 📊 참고 지표 섹션** — 접힌 채로 **`과열 N` 배지 숫자만** 본다.
  - N = 0: 넘어간다.
  - N ≥ 1이고 내 보유 비중이 큰 종목이 걸렸을 것 같으면: 펼쳐서 red/amber/blue 단계를 확인한다. **확인까지만.** 화면에도 적혀 있다: "실행할 주문은 실행 큐를 기준으로 하세요."
- [ ] **2-3. 사고 싶은 종목이 생겼다면** — 다음 분기로 처리한다:
  - 위성(투더문) 성격의 강한 추세 자산 → 관심종목에서 🐢 터틀 후보로 지정하고 **실행 큐가 신호를 줄 때까지 기다린다.**
  - 코어 성격의 장기 보유 자산 → 반기 리밸런싱 때 카테고리 비중과 ③ 대표 매수 종목으로 반영한다.
  - 둘 다 아니면 → 사지 않는다.

---

## 3. 주 1회 30분 — 신호 리플레이 복기

신호를 더 잘 믿기 위한(또는 덜 믿기 위한) 근거를 쌓는 시간. 상세 조작법은 [사용설명서 §6~§13](사용설명서_구루신호엔진_신호리플레이.md) 참고.

- [ ] **3-1.** 신호 리플레이 탭에서 이번 주 종목 1개 선택 (보유·관심 종목 칩 활용, 기간 1년).
- [ ] **3-2.** [이전 신호]/[다음 신호] 버튼으로 신호일만 훑으며 판정을 남긴다: `적절함 / 너무 빠름 / 너무 늦음 / 잘못된 신호`.
- [ ] **3-3.** 신호가 없었던 큰 상승/하락 구간이 보이면 `놓친 매수 / 놓친 매도`로 기록 + 한 줄 메모.
- [ ] **3-4.** (선택) 샌드박스에서 **조건 딱 1개만** 바꿔 본다. 좋아 보이면 연구용 사례로 저장.
  - 샌드박스 값은 라이브 신호에 반영되지 않는다. 검증용(holdout) 사례에 맞춰 고치지 않는다.

---

## 4. 월 1회 10분 — 정비

- [ ] **4-1.** 관심종목에서 🐢 뱃지가 **위성 9종목에만** 붙어 있는지 확인 (코어에 붙어 있으면 해제).
- [ ] **4-2.** 총자산이 크게 변했으면 실행 큐의 "터틀(위성) 예산"을 [총자산의 10%로 설정] → [저장]으로 갱신.
- [ ] **4-3.** 신호 리플레이의 **놓친 매수/매도 모아보기**를 훑는다 — 같은 종목·같은 조건에서 반복해 놓치고 있으면 그게 개선 1순위 후보다.

---

## 5. 반기 1회 1시간 — 코어 리밸런싱

이 루틴에서만 코어 자산을 사고판다. 리밸런싱 화면("포트폴리오 리밸런싱 (2단 배분표)")에서 진행.

- [ ] **5-1.** 맨 위 **목표 총 자산 (KRW)**을 현재 총자산으로 갱신.
- [ ] **5-2.** **① 전략 배분** — 코어 90 / 투더문 10 유지 (합 100 확인).
- [ ] **5-3.** **② 코어 자산배분** — 카테고리별 목표비중을 점검·조정 (합계 100%). 반기마다 조금씩 조정하는 곳이 여기다.
- [ ] **5-4.** **③ 대표 매수 종목 지정** — 각 코어 카테고리에 종목이 지정돼 있는지 확인.
  - `미지정`인 카테고리가 있으면 [지정]을 눌러 티커·거래소를 입력한다.
  - **미지정 상태로 두면 그 카테고리가 부족해도 매수 주문이 생성되지 않고 `대표 매수 종목 미지정`으로 스킵된다.** ③이 반기 자동화의 전제 조건인 이유.
  - 매도는 대표 종목이 필요 없다 (초과 카테고리의 보유 종목에서 생성).
- [ ] **5-5.** 오른쪽 위 **[저장하기]** → "저장됨!" 확인. (①②③ 모두 저장하기를 눌러야 반영된다.)
- [ ] **5-6.** 리밸런싱 주문 생성 → 결과를 읽는다:
  - `N건의 주문이 생성되었습니다` → 실행 큐로 가서 루틴 1처럼 실행.
  - `0건` → 사유를 확인한다. **전부 밴드 안(±5%p)이면 "주문 없음"이 정상**이다.
- [ ] **5-7.** 0건인데 밴드를 벗어난 카테고리가 있다면 스킵 사유를 읽는다:

| 스킵 사유 | 뜻 | 조치 |
|---|---|---|
| 대표 매수 종목 미지정 | ③에서 종목을 안 정했다 | 5-4로 돌아가 지정 |
| 가격 확보 실패 | 시세를 못 불러왔다 | 티커·거래소 확인, 나중에 재시도 |
| 환율 미지원(원화 환산 불가) | 해당 통화 환율 없음 | 종목/거래소 재검토 |
| 매도할 코어 보유 없음 | 초과인데 팔 보유가 없다 | 분류(카테고리) 확인 |
| 조정액이 1주 미만 | 편차가 너무 작다 | 정상 — 조치 불필요 |
| 이미 대기 중 주문 있음 | 중복 방지 | 실행 큐의 기존 주문부터 처리 |

## 6. 상황별 대응표

| 상황 | 하는 일 | 하지 않는 일 |
|---|---|---|
| 구루 매도 경고가 보유 종목에 떴다 | [왜 떴나]로 근거 확인, 메모. 실행 큐·매도 경보에 후속이 뜨는지 이후 며칠 주시 | 그 자리에서 전량 매도 |
| 실행 큐에 터틀 신규 매수가 떴다 | 카드의 수량·가격대로 체결하고 [실행 완료] | 수량을 임의로 2배로 늘리기 |
| 실행 큐에 손절 매도가 떴다 | **그날 실행한다.** 손절은 미루는 순간 시스템이 아니다 | "반등하면 팔지" 하며 스누즈 반복 |
| 참고 지표 과열 배지가 red | 펼쳐서 어느 종목인지 확인, 끝 | 과열만 보고 매도 주문 내기 |
| 아무 신호도, 주문도 없다 | "왜 주문이 없나요?" 정상 문구 확인 후 앱 종료 | 심심해서 수동 매매 |
| 신호가 이상하다(늦다/틀리다) 싶다 | 주말 리플레이에서 그 구간을 판정으로 기록 | 즉흥적으로 규칙 불신·수동 매매 |
| OHLCV·N 부족 문구에 위성 종목이 떴다 | Claude에게 보고 (코드 문제) | 신호가 안 뜬다고 수동 진입 |

## 7. 하지 말 것 (12개월 동결 규칙 포함)

1. **파라미터 동결** — 유닛 올리기, 위성 비중 확대(10%→20~30%), 전 종목 터틀, 코어를 터틀로 돌리기: 앞으로 6~12개월 금지. 유닛을 **내리는 것**(2→1)만 예외.
2. **참고 지표·구루 신호로 직접 매매하지 않는다.** 행동은 실행 큐만.
3. **샌드박스 조정값이 실전에 반영된다고 착각하지 않는다.** 반영은 별도 개발 단계다.
4. **유선 계정 자산에는 전략(터틀·리밸런싱)을 적용하지 않는다.** 유선은 상시 전략 제외(매도 경보만 유지).
5. **적중률 표본이 적으면(발화 수 한 자리) 결론 내리지 않는다.**
6. **실행 큐 주문을 사유 없이 방치하지 않는다.** 실행하거나, 사유를 적고 건너뛴다.

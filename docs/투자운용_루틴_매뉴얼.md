# Asset Manager 실전 투자 플레이북

> 구루 신호 엔진, 참고 지표, 신호 리플레이, 대표 매수 종목, 실행 큐를 하나의 투자 루틴으로 연결하는 따라 하기 매뉴얼.
>
> 앱 기준일: 2026-07-14

이 플레이북의 목표는 더 많은 신호를 찾는 것이 아니다. **관찰할 때, 연구할 때, 설정할 때, 실제로 주문할 때를 분리하여 충동적인 결정을 줄이는 것**이다.

> **한 문장 원칙: 구루 신호와 참고 지표는 관찰, 리플레이는 연구, 대표 종목은 설정, 실제 매매는 실행 큐다.**

이 문서는 앱 사용법과 운용 규율을 설명한다. 수익을 보장하거나 특정 종목의 매수·매도를 권유하지 않으며, 실제 투자 결정과 체결의 책임은 사용자에게 있다.

## 0. 30초 만에 이해하는 전체 지도

### 기능별 권한

| 기능 | 권한 | 하는 일 | 하지 않는 일 |
|---|---|---|---|
| **구루 신호 엔진** | 🟡 관찰 | 현재 보유·관심 종목 중 조건에 맞는 후보를 찾는다 | 직접 주문을 만들지 않는다 |
| **참고 지표** | 🟡 경고 | 과열 리스크를 한 번 더 확인한다 | 고점이나 폭락 시점을 예측하지 않는다 |
| **신호 리플레이** | 🔵 연구 | 현재 규칙을 과거 가격에 적용하여 타이밍을 복기한다 | 라이브 규칙이나 실제 주문을 바꾸지 않는다 |
| **③ 대표 매수 종목** | ⚙️ 설정 | 코어 카테고리가 부족할 때 살 종목을 예약한다 | 지정만으로 주문을 만들지 않는다 |
| **실행 큐** | 🔴 실행 | 종목·수량·기준가·근거가 정해진 주문을 처리한다 | 새로운 투자 아이디어를 연구하지 않는다 |

### 앱을 쓰는 선순환

```text
관찰                 연구                   설정                 실행                 복기
대시보드        →     신호 리플레이     →    대표 매수 종목   →    실행 큐        →     주간 기록
구루/참고 지표        왜 떴나·맞았나          코어가 부족하면        실제 체결값 입력       반복 실수 확인
```

이 흐름을 거꾸로 사용하지 않는다. 예를 들어 참고 지표에서 과열 경고가 보였다는 이유만으로 바로 매도하거나, 리플레이에서 좋아 보인 숫자를 곧바로 라이브 규칙으로 믿지 않는다.

### 권장 시간표

| 주기 | 시간 | 미션 |
|---|---:|---|
| 매일 | 8분 | 대시보드 관찰 3분 + 실행 큐 5분 |
| 매주 | 30분 | 종목 1개 신호 리플레이 |
| 매월 | 15분 | 기록·예산·후보 정비 |
| 반기 | 60분 | 코어 목표 점검과 리밸런싱 주문 생성 |

## 1. 첫날 미션 — 시스템을 출발선에 세우기

처음 한 번만 제대로 설정하면 이후에는 정해진 루틴만 반복하면 된다. 상세 화면 설명은 [앱 운용 설정 따라하기 매뉴얼](운용설정_따라하기_매뉴얼.md)을 참고한다.

### 미션 1 — 돈의 역할을 나눈다

리밸런싱 화면의 `포트폴리오 리밸런싱 (2단 배분표)`에서 다음 항목을 확인한다.

- `목표 총 자산 (KRW)` 입력
- `① 전략 배분 — 코어 vs 투더문` 비중 설정
- `② 코어 자산배분 — 카테고리별` 목표 비중 설정
- 모든 비중의 합계 확인
- `저장하기`를 누르고 `저장됨!` 확인

**코어**는 자산배분과 반기 리밸런싱으로 관리한다. **투더문(위성)**은 터틀 후보와 실행 큐로 관리한다. 같은 종목을 두 전략의 지시에 따라 동시에 움직이지 않는 것이 핵심이다.

계정 축을 쓰는 경우 한 가지 더: **유선 계정 자산은 코어·위성 어느 전략에도 넣지 않는다.** 유선은 상시 전략 제외이고 매도 경보만 유지된다. 통합 뷰에서 비중을 볼 때 이 부분을 혼동하지 않는다.

### 미션 2 — 코어의 장보기 목록을 만든다

`③ 대표 매수 종목 지정 (코어 리밸런싱 매수용)`에서 각 코어 카테고리가 부족할 때 살 종목을 지정한다.

좋은 대표 종목의 실무 조건은 다음과 같다.

- 해당 코어 카테고리의 목적을 대표한다.
- 실제 사용하는 증권사에서 거래할 수 있다.
- 티커와 거래소가 정확하다.
- 장기간 유지할 수 있어 반기마다 바꿀 필요가 없다.
- 같은 카테고리 안에서 무엇을 살지 미리 결정하여 당일 감정을 끼워 넣지 않는다.

지정 또는 변경 후 반드시 상단의 `저장하기`를 누른다. 미저장 변경이 있으면 리밸런싱 주문 생성 버튼이 비활성화된다.

> 대표 매수 종목은 “좋아 보이는 종목 추천”이 아니다. **해당 카테고리가 목표보다 부족할 때 사용할 기본 도구**다.

### 미션 3 — 위성의 운동장을 정한다

실행 큐 상단의 `터틀(위성) 예산`을 설정하고, 관심종목에서 터틀 전략을 적용할 위성 후보에만 🐢 표시를 붙인다.

- 코어 자산에는 터틀 후보를 켜지 않는다.
- 위성 예산이 0이면 신규 진입 주문이 생성되지 않는다.
- 후보가 없으면 터틀 진입을 평가할 종목도 없다.
- 시세가 갱신되지 않은 후보는 평가에서 건너뛸 수 있다.

### 첫날 완료 체크

- [ ] 목표 총 자산과 코어/투더문 비중을 저장했다.
- [ ] 코어 카테고리 비중의 합계를 확인했다.
- [ ] 각 코어 카테고리의 대표 매수 종목을 지정하고 저장했다.
- [ ] 터틀 위성 예산을 저장했다.
- [ ] 위성 후보에만 🐢 표시를 붙였다.
- [ ] 코어와 위성을 어떤 원칙으로 관리할지 구분할 수 있다.

## 2. 매일 8분 미션 — 보고, 만들고, 처리한다

매일 같은 시간대에 한 번만 점검하는 것을 권장한다. 앱의 시세 데이터가 갱신된 뒤 실행해야 당일 판단의 기준이 일관된다.

### 0:00~3:00 — 대시보드 관찰

#### 1) 구루 신호 엔진은 매도 경고부터 본다

대시보드 상단의 `🧭 구루 신호 엔진`을 연다.

1. 보유 종목의 매도 경고가 있는지 확인한다.
2. 경고가 있으면 `왜 떴나`를 열어 충족 조건을 읽는다.
3. 관찰 후보나 진입 검토 후보도 같은 방식으로 근거를 확인한다.
4. 아무 신호가 없는데 이유가 궁금하면 `왜 신호가 안 뜨나요?`를 연다.

신호가 없다는 것은 오류가 아니라 조건 미충족일 수 있다. 데이터 부족과 조건 미충족을 진단에서 구분한다.

**오늘 할 일:** 근거 확인, 메모, 관심종목 정리.

**오늘 하지 않을 일:** 신호 카드만 보고 즉시 매수·전량 매도.

#### 2) 참고 지표는 배지만 먼저 본다

구루 카드 아래의 접힌 `📊 참고 지표`에서 `과열 N` 배지를 확인한다.

- `과열 0`: 그대로 닫고 지나간다.
- `과열 1 이상`: 펼쳐서 어떤 종목인지, 위험 단계가 무엇인지 확인한다.
- 내 포트폴리오에서 비중이 큰 종목이면 주간 리플레이 후보나 점검 메모에 올린다.

과열은 “곧 폭락한다”는 뜻이 아니다. 가격이 계속 오르면서 과열 상태가 오래 유지될 수도 있다. 따라서 과열 배지는 **속도를 줄이고 근거를 다시 보라는 노란 신호등**이지 매도 버튼이 아니다.

#### 3) 관찰 결과를 코어와 위성으로 분류한다

| 발견한 종목 | 다음 행동 |
|---|---|
| 장기 자산배분용 코어 | 대표 매수 종목과 반기 리밸런싱에서 다룬다 |
| 강한 추세를 추적할 위성 후보 | 관심종목에서 검토 후 🐢 후보로 지정한다 |
| 정체가 불명확한 아이디어 | 바로 매수하지 않고 주간 리플레이 목록에 넣는다 |

### 3:00~8:00 — 실행 큐 처리

실행 큐의 화면 제목은 `오늘의 주문서`다. 이곳의 카드는 신호가 아니라 **처리를 기다리는 주문**이다.

#### 1) `오늘 주문 생성`을 누른다

이 버튼은 현재 시점의 터틀 규칙을 평가한다. 화면에 들어오기만 해서는 새 터틀 주문이 생기지 않는다.

자동 생성을 원한다면 `오늘 주문 자동 생성`을 명시적으로 켤 수 있다. 기본값은 꺼짐이며, 켜면 앱 시작 후 시세 업데이트가 끝났을 때 하루 한 번 평가한다.

> 중요: `오늘 주문 생성`은 터틀 주문용이다. **코어 리밸런싱 주문은 리밸런싱 화면의 `리밸런싱 주문 생성`에서 별도로 만든다.**

#### 2) 주문 카드를 읽는다

| 주문 종류 | 의미 | 기본 태도 |
|---|---|---|
| `TURTLE_ENTRY` | 55일 신고가 돌파 신규 매수 | 수량·기준가·근거와 실제 체결 가능 여부 확인 |
| `TURTLE_PYRAMID` | 상승 중 추가 매수 | 기존 포지션과 전체 위성 위험 확인 |
| `TURTLE_STOP` | 2N 손절 전량 매도 | 우선순위를 가장 높게 두고 지연을 최소화 |
| `TURTLE_EXIT` | 20일 최저가 청산 | 규칙에 따른 추세 종료로 처리 |
| `REBALANCE_BUY` | 부족한 코어 카테고리 매수 | 저장된 목표와 대표 종목 확인 |
| `REBALANCE_SELL` | 초과한 코어 카테고리 매도 | 세금·계좌·실제 수량을 최종 확인 |
| `CLEANUP_SELL` | 대청소 청산 | 정리 대상과 근거를 다시 확인 |

카드에서 종목, 수량, 기준가, 생성 근거를 읽고 증권사의 현재 호가와 비교한다. 기준가는 생성 시점의 참고값이며 실제 체결가는 다를 수 있다.

#### 3) 세 가지 중 하나를 선택한다

- `실행하기`: 증권사에서 체결한 뒤 실제 체결일·체결가·수량을 입력한다.
- `건너뜀`: 실행하지 않는 이유를 반드시 적는다. 예: 계좌 현금 부족, 이미 수동 처리함.
- `내일`: 하루 미룬다. 연속 2회부터 주의, 연속 3회 이상 또는 7일 이상 방치는 경고로 강조된다. 습관적인 연기 수단으로 쓰지 않는다.

실행하지 못했다면 침묵하지 말고 `건너뜀` 또는 `내일`로 의사결정을 남긴다. 기록이 있어야 월간 복기에서 시스템 문제와 실행 문제를 구분할 수 있다.

#### 4) 주문이 0건이면 진단한다

`왜 주문이 없나요?` 패널의 문장을 두 종류로 읽는다.

**조치가 필요한 경우**

- 위성 예산이 0이다 → 예산을 설정한다.
- 터틀 후보가 없다 → 관심종목에서 후보를 지정한다.
- 시세가 갱신되지 않았다 → 관심종목 시세를 갱신한다.
- OHLCV 또는 N이 부족하다 → 데이터 확보 상태를 확인한다. 위성 후보인데 이 문구가 계속 뜨면 그 종목은 터틀 신호가 영영 안 뜨는 계기판 고장이다. 투자 판단이 아니라 코드 문제이므로 개발(Claude)에 보고한다.

**기다리는 것이 정상인 경우**

- 55일 신고가를 돌파하지 않았다.
- 손절·청산·불타기 조건이 충족되지 않았다.
- 동시 리스크 한도를 넘었다.
- 남은 예산으로 1유닛을 살 수 없다.
- 같은 종목의 주문이 이미 대기 중이다.

> 주문 0건은 실패가 아니다. 조건이 없을 때 아무것도 하지 않는 것도 시스템의 결정이다.

### 매일 완료 체크

- [ ] 구루 신호의 매도 경고부터 확인했다.
- [ ] `과열 N` 배지를 확인했다.
- [ ] 신호를 보고 즉흥 주문을 내지 않았다.
- [ ] `오늘 주문 생성`을 실행했거나 자동 생성 결과를 확인했다.
- [ ] 대기 주문을 실행·건너뜀·내일 중 하나로 처리했다.
- [ ] 주문 0건이면 정상 대기와 조치 필요를 구분했다.

## 3. 주간 30분 미션 — 신호 탐정 훈련

신호 리플레이의 목적은 “가장 수익률이 높은 숫자”를 찾는 것이 아니다. **내가 어떤 신호를 과신하는지, 어떤 좋은 구간을 반복해서 놓치는지 증거를 모으는 것**이다.

기능의 전체 설명은 [구루 신호 엔진 & 신호 리플레이 사용 설명서](사용설명서_구루신호엔진_신호리플레이.md)를 참고한다.

### 한 번의 복기 순서

#### 1) 이번 주의 종목 하나를 고른다 — 3분

우선순위는 다음과 같다.

1. 실제로 실행하거나 건너뛴 주문이 있었던 종목
2. 구루 신호와 과열 경고가 겹친 종목
3. 신호가 이상하다고 느낀 종목
4. 새로 터틀 후보에 넣고 싶은 종목

검색창 또는 보유·관심 종목 칩을 이용한다. 처음에는 한 번에 한 종목만 깊게 본다.

#### 2) 먼저 `리플레이(미래 숨김)`으로 본다 — 7분

미래를 모르는 상태에서 날짜를 이동하며 다음을 적는다.

- 이 시점에서 실제로 진입하고 싶었는가?
- 손절 위치와 감당 가능한 손실을 설명할 수 있는가?
- 구루 진단에서 정확히 어떤 조건이 충족됐는가?
- 가격기반 알림은 리플레이에서 검증 가능한 범위인가?

`이전 신호`와 `다음 신호`는 구루 신호 발생일 사이를 이동한다. RSI·골든크로스·데드크로스 등은 차트 마커가 아니라 `가격기반 알림 진단`에서 확인한다.

#### 3) `복기(미래 공개)`로 결과를 확인한다 — 7분

미래 차트를 공개한 뒤 다음 세 가지를 본다.

- 신호가 방향은 맞았지만 너무 빨랐거나 늦었는가?
- 신호 후 상승·하락이 일시적이었는가, 추세로 이어졌는가?
- 거래량, 이동평균선, RSI가 신호의 품질을 설명하는가?

리플레이는 현재 활성 규칙을 과거 가격에 다시 적용한 결과다. **그날 앱이 실제로 보낸 알림 기록은 아니다.**

#### 4) `내 판정`을 남긴다 — 5분

| 판정 | 사용하는 때 |
|---|---|
| `적절함` | 실행 가능한 시점에 의미 있는 신호였다 |
| `너무 빠름` | 방향은 맞을 수 있으나 확인이 부족했다 |
| `너무 늦음` | 움직임의 상당 부분이 끝난 뒤였다 |
| `잘못된 신호` | 후속 움직임이 신호의 의미를 지지하지 않았다 |
| `놓친 매수` | 좋은 매수 구간인데 신호가 없었다 |
| `놓친 매도` | 피할 만한 하락 구간인데 신호가 없었다 |

가능하면 짧은 메모를 붙인다.

```text
관찰: MA20 위였지만 거래량 증가가 없었음
판정: 너무 빠름
다음 확인: 거래량 동반 사례와 비교
```

날짜 전체뿐 아니라 특정 구루 규칙을 선택해 규칙별 판정을 남길 수 있다.

#### 5) 한 줄 결론만 쓴다 — 3분

다음 템플릿을 사용한다.

> 이번 종목에서 `[규칙명]`은 `[어떤 시장 상태]`에서는 유용했지만, `[어떤 조건]`에서는 오탐이 많았다. 다음 주에는 `[비교할 사례]`를 확인한다.

한 사례만 보고 규칙을 바꾸지 않는다. 다음 주 비교 질문 하나를 남기는 것으로 끝낸다.

#### 6) 기록을 백업한다 — 필요 시 5분

판정과 검증 사례는 브라우저 저장소에 보관된다. 중요한 기록은 화면 하단의 `내보내기`로 JSON 백업한다. 오래된 기록을 정리하기 전에도 먼저 내보내는 것이 안전하다.

### 4주 훈련 코스

| 주차 | 미션 | 성공 조건 |
|---|---|---|
| 1주차 | 마커와 `왜 떴나/안 떴나` 읽기 | 신호의 충족 조건을 말로 설명한다 |
| 2주차 | 큰 상승 구간의 매수 신호 평가 | 적절함·빠름·늦음을 구분한다 |
| 3주차 | 큰 하락 구간의 매도 경고 평가 | 경고와 실제 청산 주문을 구분한다 |
| 4주차 | 놓친 매수·매도와 오탐 비교 | 다음에 검증할 가설 하나를 만든다 |

### 샌드박스 사용 규율

규칙 샌드박스에서는 한 번에 조건 하나만 바꾼다.

1. 기준 규칙의 결과를 기록한다.
2. 조건 하나의 값 또는 on/off만 변경한다.
3. 좋아진 사례뿐 아니라 나빠진 사례도 확인한다.
4. 다른 종목과 다른 시장 구간에서 다시 확인한다.
5. 검증 사례를 저장한다.

샌드박스 값은 미리보기이며 라이브 구루 신호·알림·저장된 규칙을 바꾸지 않는다. 유리한 구간 하나에 맞춘 값은 발견이지 결론이 아니다.

## 4. 월간 15분 미션 — 시스템 건강검진

월말에는 수익률보다 **내가 시스템을 제대로 실행했는지**를 먼저 본다.

### 1) 실행 규율 점검

- 실행 큐 주문을 이유 없이 방치한 적이 있는가?
- `내일`을 반복한 주문은 무엇인가?
- `건너뜀` 사유가 같은 이유로 반복되는가?
- 실제 체결가·수량을 정확히 기록했는가?
- 손절 주문을 다른 주문보다 늦게 처리하지 않았는가?

반복되는 건너뜀은 의지 부족이 아니라 예산, 계좌, 후보 구성 또는 생활 리듬의 문제일 수 있다.

### 2) 위성 설정 점검

- 총자산이 크게 변했다면 터틀 위성 예산을 다시 계산해 저장한다.
- 🐢 후보가 여전히 위성 전략에 적합한지 확인한다.
- 코어 자산에 🐢가 잘못 붙어 있으면 해제한다.
- 가격 데이터가 반복해서 부족한 종목은 데이터 경로를 점검한다.

### 3) 연구 기록 점검

- `놓친 매수/매도 모아보기`에서 반복 패턴을 찾는다.
- 같은 규칙에 `잘못된 신호`가 누적되는지 본다.
- 표본이 적은데 성과 숫자만 보고 결론 내리지 않았는지 확인한다.
- 다음 달에 집중할 규칙 또는 종목을 하나만 고른다.

### 월간 점수판

수익률이 아니라 과정에 점수를 준다.

| 항목 | 1점 조건 |
|---|---|
| 관찰 | 정한 날의 대시보드 점검을 빠뜨리지 않았다 |
| 실행 | 모든 대기 주문에 결정을 남겼다 |
| 기록 | 주간 판정과 메모를 남겼다 |
| 분리 | 참고 신호로 직접 주문하지 않았다 |
| 동결 | 단기 성과 때문에 설정을 즉흥 변경하지 않았다 |

`4~5점`: 루틴 유지.

`2~3점`: 시간을 줄이더라도 핵심 루틴부터 복구.

`0~1점`: 새 전략을 추가하지 말고 실행 가능한 구조부터 다시 설계.

## 5. 반기 60분 미션 — 코어 리밸런싱 데이

코어 자산을 사고파는 공식 점검 시간이다. 평소 구루 신호나 과열 경고로 코어 비중을 흔들지 않고, 반기 점검에서 저장된 목표와 밴드를 사용한다.

### 1단계 — 목표를 검토한다

1. `목표 총 자산 (KRW)`을 현재 계획에 맞게 갱신한다.
2. `① 전략 배분 — 코어 vs 투더문`을 검토한다.
3. `② 코어 자산배분 — 카테고리별` 목표를 검토한다.
4. 변경했다면 `저장하기`를 누른다.

단기 시장 전망만으로 목표 비중을 바꾸지 않는다. 목표 변경에는 생활자금, 투자 기간, 감당 가능한 변동성처럼 장기적인 이유가 있어야 한다.

### 2단계 — 대표 매수 종목을 점검한다

`③ 대표 매수 종목 지정`에서 각 카테고리를 확인한다.

- 부족한 카테고리: 지정된 대표 종목으로 매수 주문을 만들 수 있다.
- 초과한 카테고리: 실제 보유 종목을 기준으로 매도 주문을 만든다.
- 대표 종목 미지정: 해당 카테고리의 매수 주문은 스킵된다.

변경했다면 다시 `저장하기`를 누른다.

### 3단계 — 밴드를 읽는다

현재 자동 주문 대상은 **`② 코어 카테고리`의 목표 대비 ±5%p 밴드 이탈**이다.

- `① 전략 배분(코어/투더문)` 이탈 자체는 현재 주문 생성 대상이 아니다.
- 모든 코어 카테고리가 밴드 안이면 주문 0건이 정상이다.
- 밴드 이탈 안내는 먼저 보여주는 참고 정보이고, 실제 종목·수량은 다음 단계에서 계산된다.

### 4단계 — `리밸런싱 주문 생성`을 누른다

저장된 목표를 기준으로 주문을 생성한다. 결과가 생성되면 실행 큐로 이동한다.

| 결과 | 의미 | 조치 |
|---|---|---|
| 주문 N건 생성 | 실행 가능한 리밸런싱 카드가 큐에 추가됨 | 실행 큐에서 실제 체결값 입력 |
| 모두 밴드 안 | 지금 조정할 것이 없음 | 정상 종료 |
| 대표 매수 종목 미지정 | 부족한 카테고리의 매수 도구 없음 | ③에서 지정 후 저장 |
| 가격 확보 실패 | 종목 가격을 가져오지 못함 | 티커·거래소·시세 확인 |
| 환율 미지원 | 원화 조정액을 종목 통화로 계산 불가 | 종목 통화·환율 확인 |
| 매도할 코어 보유 없음 | 초과 카테고리에 실제 매도 대상이 없음 | 자산 분류 확인 |
| 조정액이 1주 미만 | 필요한 조정 규모가 너무 작음 | 정상 대기 |
| 이미 대기 중 주문 있음 | 중복 생성을 차단함 | 기존 실행 큐 주문부터 처리 |

### 5단계 — 실행 큐에서 마무리한다

`REBALANCE_BUY`와 `REBALANCE_SELL` 카드의 목표, 종목, 수량을 다시 확인한다. 증권사에서 체결한 뒤 `실행하기`를 통해 실제 체결일·체결가·수량을 입력한다.

세금, 거래 가능 시간, 계좌별 보유 상태는 앱 바깥의 실제 조건이므로 주문 전 최종 확인한다.

### 반기 완료 체크

- [ ] 목표 총 자산과 비중을 검토하고 저장했다.
- [ ] 대표 매수 종목의 티커와 거래소를 확인했다.
- [ ] 미저장 변경이 없는 상태에서 주문을 생성했다.
- [ ] 주문 0건의 이유를 정상·오류·미설정으로 구분했다.
- [ ] 생성된 주문을 실행 큐에서 실제 체결값으로 처리했다.
- [ ] 단기 전망 때문에 목표를 즉흥적으로 바꾸지 않았다.

## 6. 실전 상황 카드

### 상황 A — 구루 매도 경고와 과열 경고가 동시에 떴다

**올바른 행동**

1. `왜 떴나`에서 각각의 조건을 확인한다.
2. 보유 목적이 코어인지 위성인지 구분한다.
3. 위성이면 실행 큐의 손절·청산 주문을 확인한다.
4. 코어이면 반기 리밸런싱 원칙과 밴드를 유지한다.
5. 주간 리플레이 후보로 기록한다.

**피할 행동:** 두 경고가 겹쳤다는 이유만으로 즉시 전량 매도.

### 상황 B — 구루 매수 후보는 있는데 실행 큐 주문이 없다

**해석:** 구루 카드는 관찰 후보이고 실행 큐의 터틀 진입 조건과는 다른 체계다.

**올바른 행동:** 위성 후보로 적합한지 검토하고, 🐢 지정 여부·예산·55일 돌파·데이터 상태를 확인한다. 조건이 충족될 때까지 기다린다.

### 상황 C — 실행 큐에 터틀 신규 매수가 떴다

**올바른 행동:** 카드의 근거, 수량, 기준가, 위성 예산을 확인하고 실제 체결 후 값을 입력한다.

**피할 행동:** 확신이 강하다는 이유로 수량을 임의로 두 배로 늘린다.

### 상황 D — 실행 큐에 손절 주문이 떴다

**올바른 행동:** 다른 관찰 신호보다 우선하여 처리한다. 실행하지 못했다면 이유를 기록한다.

**피할 행동:** 반등을 기다리며 `내일`을 반복한다.

### 상황 E — 리밸런싱 주문이 한 건도 없다

**확인 순서**

1. 목표 총 자산과 코어 비중이 저장됐는가?
2. 코어 카테고리가 ±5%p 밴드를 벗어났는가?
3. 부족한 카테고리에 대표 매수 종목이 있는가?
4. 가격·환율·보유 데이터가 있는가?
5. 같은 주문이 이미 실행 큐에 있는가?

모두 밴드 안이라면 아무것도 하지 않는 것이 정답이다.

### 상황 F — 리플레이에서 놀라운 설정값을 찾았다

**올바른 행동:** 기준 결과를 저장하고, 다른 시기·다른 종목·불리한 구간에서 반복 검증한다.

**피할 행동:** 한 종목의 가장 좋은 구간에 맞춰 파라미터를 계속 조정한다.

### 상황 G — 참고 지표는 위험한데 가격은 계속 오른다

**해석:** 과열 경고는 꼭지 예측이 아니다.

**올바른 행동:** 신규 추격 여부를 더 신중히 보고, 기존 포지션은 정해 둔 실행 규칙으로 관리한다.

### 상황 H — 며칠 동안 앱을 열지 못했다

**해석:** 주문은 사라지지 않는다. 실행 큐는 Drive에 저장되어 처리할 때까지 대기하며, 밀린 주문은 에스컬레이션 표시로 강조된다.

**올바른 행동:** `오늘 주문 생성`으로 오늘 기준으로 다시 평가한 뒤, 에스컬레이션된 주문부터 처리한다. 그중에서도 손절·청산이 최우선이다.

**피할 행동:** 밀렸다는 이유로 대기 주문을 사유 없이 계속 방치한다.

## 7. 12개월 운용 규율

1. **기능의 권한을 섞지 않는다.** 참고는 참고, 연구는 연구, 주문은 실행 큐다.
2. **코어와 위성을 섞지 않는다.** 코어는 리밸런싱, 위성은 터틀 실행 큐로 관리한다.
3. **손절을 토론거리로 만들지 않는다.** 실행 큐에 생성된 손절은 우선 처리한다.
4. **주문이 없는 날을 견딘다.** 신호를 만들기 위해 후보나 조건을 억지로 바꾸지 않는다.
5. **한 사례로 규칙을 바꾸지 않는다.** 여러 종목과 시장 구간에서 반복성을 확인한다.
6. **샌드박스를 라이브 설정으로 착각하지 않는다.** 샌드박스는 연구 메모다.
7. **대표 매수 종목을 자주 교체하지 않는다.** 카테고리 목적이 변한 경우에만 재검토한다.
8. **미실행을 숨기지 않는다.** 건너뜀 사유와 연기 기록을 남긴다.
9. **실제 체결값을 기록한다.** 기준가와 체결가는 다를 수 있다.
10. **성과보다 과정부터 평가한다.** 한 달 수익률보다 정해진 루틴 준수 여부를 먼저 본다.

### 백테스트로 확정된 파라미터의 동결 목록

앞으로 6~12개월 동안 아래 변경은 하지 않는다. 성과가 좋아 보여도, 나빠 보여도 마찬가지다.

- **금지**: 유닛 확대(2→3 이상), 위성 비중 확대(10% → 20~30%), 전 종목 터틀 적용, 코어 자산(니케이·QQQ 등)의 터틀 전환.
- **유일한 예외**: 연패가 정말 견디기 힘들 때 유닛 축소(2→1). 시스템을 포기하는 것보다 낫다.

1년 뒤, 실행 큐에 쌓인 실제 실행·건너뜀 기록과 체감을 근거로 재판단한다.

## 9. 인쇄용 한 장 체크리스트

### 매일 8분

- [ ] 구루 신호의 매도 경고와 `왜 떴나` 확인
- [ ] 참고 지표의 `과열 N` 확인
- [ ] 관찰 결과를 코어·위성·연구 후보로 구분
- [ ] 실행 큐에서 `오늘 주문 생성`
- [ ] 모든 대기 주문에 실행·건너뜀·내일 결정
- [ ] 0건이면 `왜 주문이 없나요?` 확인

### 매주 30분

- [ ] 이번 주 종목 하나 선택
- [ ] 미래 숨김 모드에서 먼저 판단
- [ ] 미래 공개 후 결과 비교
- [ ] 판정 1개와 메모 저장
- [ ] 다음 주 검증 질문 하나 작성

### 매월 15분

- [ ] 반복 건너뜀·연기 사유 확인
- [ ] 위성 예산과 🐢 후보 확인
- [ ] 놓친 매수·매도 반복 패턴 확인
- [ ] 중요한 판정·사례 JSON 백업

### 반기 60분

- [ ] 목표 총 자산과 전략 비중 검토
- [ ] 코어 카테고리 비중 검토
- [ ] 대표 매수 종목과 거래소 확인
- [ ] 저장 후 리밸런싱 주문 생성
- [ ] 생성 결과 또는 0건 사유 확인
- [ ] 실행 큐에서 실제 체결값 입력

## 10. 마지막으로 기억할 세 문장

> **신호가 많다고 좋은 투자가 되는 것은 아니다. 역할이 분명한 신호가 좋은 신호다.**

> **주문이 없는 날은 시스템이 쉬는 날이지, 내가 실패한 날이 아니다.**

> **이 앱의 진짜 장점은 미래를 맞히는 데 있지 않고, 같은 원칙을 반복해서 실행하고 복기하는 데 있다.**

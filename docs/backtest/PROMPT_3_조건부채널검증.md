# 조건부 돌파 채널 가설 검증 — Codex 실행 프롬프트

아래 작업을 현재 `asset_manager` 저장소에서 끝까지 수행하라. 이 작업의 목적은 **좋아 보이는 조합을 찾는 것**이 아니라, 다음의 사전 지정된 가설이 실제로 재현되는지를 편향 없이 검증하는 것이다.

> 대형주 또는 업종·카테고리 대장주는 20일 신고가 돌파 진입이 유리하고, 그 밖의 종목은 55일 신고가 돌파 진입이 유리한가?

결론이 가설을 지지하지 않아도 정상적인 성공이다. 데이터가 부족하면 억지로 수치를 만들지 말고 `INCONCLUSIVE`로 종료하라.

1. 시작 전에 반드시 다음 파일을 전부 읽고, 서로 다른 규칙이 있으면 더 엄격한 규칙을 따른다.
   - `AGENTS.md`
   - `RULES.md`
   - `docs/백테스트_계획_진입채널비교.md`
   - `docs/backtest/REPORT_채널검증.md`
   - `docs/backtest/REPORT_종목별채널.md`
   - `docs/backtest/PROMPT_2_백테스트실행.md`
   - `scripts/backtest/turtlePortfolio.ts`
   - `scripts/backtest/lib/satelliteTurtle.ts`
   - `scripts/backtest/lib/calendar.ts`
   - `scripts/backtest/sectorRotation/lib/yahooData.ts`
   - `types/turtle.ts`
2. 기존 보고서는 참고 자료일 뿐 재현 가능한 원시 결과로 간주하지 않는다. 현재 저장소에는 `REPORT_채널검증.md`와 `REPORT_종목별채널.md`를 동일하게 재생성하는 커밋된 실행기가 없으므로, **결정론적 실행기·설정·데이터 명세·원시 집계 결과를 새로 구축**해야 한다. 존재하지 않는 “기존 종목별 채널 실행기”를 확장했다고 쓰지 말라.
   - `alignToCalendar()`의 carry-forward는 포트폴리오 평가액 정렬에만 사용할 수 있고 채널·ATR 계산에는 사용할 수 없다.
   - 현재 `satelliteTurtle.ts`의 장중 돌파 체결·공통 정렬 배열 가정을 확인하라. 의미가 다른 코드를 1차 종가확인/익일시가 실행기로 그대로 재사용하지 않는다.
3. 검증 코드는 앱 런타임과 분리한다. 앱의 기본 터틀 설정, UI, 포트폴리오 데이터 또는 실거래 로직은 변경하지 않는다.
4. 기존 사용자의 변경 사항과 더티 워크트리를 보존한다. 관련 없는 파일은 수정하지 않는다.
5. 코드 파일을 추가·수정했다면 저장소 규칙에 따라 `RULES.md`의 해당 파일 책임과 주의사항을 **그 즉시** 갱신한다. `any`, `console.*`, 하드코딩된 Cloud Run URL은 금지한다.
6. 사용자 승인 없이 커밋, 푸시, 배포 또는 앱 기본값 변경을 하지 않는다.

## 2. 기존 증거의 위치와 이번 검증의 성격

기존 보고서에서 대표 ETF 19개는 20/10이 11 대 8로 우세했으나 이항검정 `p=0.648`, 개인 보유군 76개는 55/20이 39 대 37로 우세했으나 `p=0.909`였고, 보정 후 채널 승자를 안정적으로 예측한 종목 특성은 없었다. 따라서 이번 가설은 기존 결론이 아니라 **후속 가설**이다.

- 기존 보고서에 사용된 종목·기간을 다시 나누는 분석은 모두 `EXPLORATORY`로 표시한다.
- 새로 확보한 point-in-time 전체 종목 패널에서 결과를 보기 전에 아래 규칙을 동결한 분석만 `PREREGISTERED REPLICATION`으로 표시한다.
- 과거 보고서와 종목·기간이 겹치면 완전히 독립적인 확증이라고 표현하지 않는다.
- 결과를 보고 분류 기준, 기간, 비용, 종목 제외 규칙 또는 채널을 바꾸면 해당 결과는 확증 분석에서 제외한다.

## 3. 0단계 — 데이터 가능성 감사와 강제 중단 조건

수익률이나 채널별 성과를 계산하기 전에 데이터 가능성 감사부터 수행한다. 다음 정보가 날짜별(point-in-time)로 필요하다.

현재 저장소 감사상 역사적 시가총액·역사적 업종 분류·상장폐지 포함 전체 종목군은 커밋된 데이터로 확인되지 않았다. 실제로 신뢰할 수 있는 외부 소스를 확보하기 전에는 “있다고 가정”하지 않는다.

| 필드 | 필수 조건 |
|---|---|
| 종목 식별자 | 티커 변경과 재상장을 구분하는 안정적인 영구 식별자 |
| 상장 정보 | 거래소, 상장일, 상장폐지일, 종목 유형, 거래정지 기간 |
| 가격 | 종목별 실제 거래일의 일별 OHLCV, 통화, 조정 근거 |
| 기업행위 | 액면분할·병합, 현금배당, 합병·인수, 상장폐지 대가 또는 상장폐지 수익률 |
| 규모 | 해당 시점의 발행주식수 또는 역사적 시가총액 |
| 분류 | 해당 시점에 유효했던 업종 분류 코드와 유효 시작·종료일 |
| 비용 | 시장·연도별 수수료와 세금, 가능하면 스프레드 또는 유동성 대용치 |
| 환율 | 시장 통합 포트폴리오를 만들 경우 거래일별 FX |

각 소스에 대해 제공자, 테이블/엔드포인트, 추출 시각, 데이터 버전, 기간, 시장, 행 수, 결측률, 중복률, 조정 방식, 라이선스 제약과 파일 체크섬을 매니페스트에 기록한다. 원천 데이터가 라이선스상 커밋 불가하면 원천 파일은 커밋하지 말고 재생성 방법과 체크섬만 남긴다. 자격증명이나 개인 정보는 절대 기록하지 않는다.

다음 중 하나라도 충족하면 **1차 확증 백테스트를 실행하지 말고** 데이터 격차 보고서와 필요한 다음 입력만 작성한다.

- 현재 종목 목록을 과거 전체 기간에 소급 적용해야 하는 경우
- 현재 시가총액이나 현재 업종을 과거 분류에 대신 써야 하는 경우
- 상장폐지·합병 종목을 포함한 point-in-time 투자 가능 종목군을 만들 수 없는 경우
- 분할 조정, 거래정지, 중복 바 또는 통화 오류를 신뢰성 있게 처리할 수 없는 경우
- 상장폐지 수익을 대량으로 0 또는 최종 종가로 임의 대체해야 하는 경우
- 룩어헤드 또는 생존편향을 제거할 수 없는 경우
- 잠금 표본이 3년 미만인 경우
- 어느 핵심 그룹이 시장별 30종목 미만이거나 잠금 표본에서 100회 미만의 청산 거래만 갖는 경우

Yahoo/Stooq 등 현재 티커 중심 무료 가격만으로는 이 가설의 1차 확증 검증을 충족한 것으로 보지 않는다. 불완전한 데이터로 보조 실험을 할 수는 있지만 제목과 결론에 `EXPLORATORY / SURVIVORSHIP-BIASED`를 명시하고 앱 변경 근거로 사용하지 않는다.

## 4. 사전등록과 잠금 절차

1. 데이터의 기간·행 수·결측 등 **수익률과 무관한 품질 정보만** 먼저 확인한다.
2. 성과를 계산하기 전에 다음 두 파일을 작성한다.
   - `docs/backtest/PREREG_조건부채널검증.md`
   - `scripts/backtest/conditionalChannel/preregistered-config.json`
3. 설정 JSON에는 모든 기간 경계, 종목 포함 규칙, 그룹 정의, 주문 규칙, 비용, 포지션 규칙, 난수 시드, 부트스트랩 횟수와 판정 기준을 담는다.
4. 정규화한 설정 JSON의 SHA-256을 사전등록 문서와 최종 보고서 양쪽에 기록한다.
5. 동결 후 설정이 바뀌면 기존 파일을 덮어쓰지 않는다. 변경 사유와 시각을 남긴 새 버전을 만들고, 변경 후 결과는 `AMENDED/EXPLORATORY`로 분리한다.
6. 새 point-in-time 패널의 공통 가용 기간을 달력 순서대로 60% 개발, 20% 검증, 20% 잠금 표본으로 나눈다. 잠금 표본은 최소 3개 완전 연도여야 한다.
7. 개발·검증 구간은 코드와 데이터 오류 확인에만 사용하고 채널이나 그룹 기준 최적화에 사용하지 않는다. 잠금 표본은 모든 코드·테스트·설정이 동결된 뒤 딱 한 번 연다.
8. 잠금 결과를 본 뒤 규칙을 바꾸면 원래 잠금 표본은 다시 잠금 표본으로 부르지 않는다.
9. 이 가설의 사전등록일은 **2026-07-15**다. 그 이전 데이터로 만든 마지막 20% 잠금 구간은 `SEALED PSEUDO-OOS` 또는 `PREREGISTERED REPLICATION`으로만 부르고 진정한 미래 홀드아웃이라고 쓰지 않는다. `PROSPECTIVE HOLDOUT`은 2026-07-15 이후 새로 발생한 데이터에만 사용한다.

## 5. 1차 투자 가능 종목군

미국과 한국을 **각각** 분석하고, 사전에 지정한 동일한 원칙으로만 합산한다.

- 1차: 미국 거래소 상장 보통주, 한국 KOSPI/KOSDAQ 상장 보통주
- 제외: ETF/ETN, 리츠, 우선주, SPAC, 펀드, 채권, 선물, 옵션, 암호자산
- 상장폐지, 합병, 티커 변경, 거래정지 종목을 포함한다.
- 월말에 존재했던 정보로 다음 달 투자 가능 종목군과 그룹을 확정한다.
- 신규 상장주는 55개 진입 채널, 20일 ATR 및 공통 시작 조건을 모두 충족하기 전까지 신호를 내지 않는다.
- 가격·거래량 필터를 적용한다면 결과를 보기 전에 시장별 절대값과 근거를 동결한다. 소형주에 불리하도록 사후 조정하지 않는다.
- 미국과 한국의 종목 수 차이가 합산 결과를 지배하지 않도록 합산 통계는 시장별 결과에 50:50 가중한다. 종목 수 가중 결과는 보조로만 제시한다.

## 6. 그룹 정의 — 미래 정보 금지

매월 말 `m`에 알 수 있었던 값으로 `m+1`월의 그룹을 정한다.

1. **대형주(`large`)**
   - 각 국가의 1차 투자 가능 종목군에서 point-in-time 총시가총액의 80백분위 이상인 종목, 즉 종목 수 기준 상위 20%.
   - 총시가총액은 그 시점의 종가 × 그 시점의 발행주식수 또는 동등한 역사적 시가총액 필드로 계산한다.
   - 80백분위 동률은 안정적 종목 식별자 오름차순으로 끊고 정확한 처리 규칙을 기록한다.
2. **카테고리 대장주(`leader`)**
   - point-in-time 외부 업종 분류(GICS/WICS 또는 데이터 제공자의 일관된 표준 분류)를 사용한다.
   - 해당 월의 투자 가능 종목이 최소 5개인 업종에서 총시가총액 1위인 종목이다.
   - 앱의 개인 자산 카테고리나 현재 업종을 과거 전체 기간에 소급하지 않는다.
3. **그룹 A**: `large OR leader`.
4. **그룹 B**: 같은 달 투자 가능 종목 중 `NOT large AND NOT leader`. 즉 A의 정확한 여집합이다.
5. 포지션 진입 시점의 그룹을 해당 거래가 끝날 때까지 유지한다. 보유 도중 월이 바뀌었다고 채널이나 그룹을 변경하지 않는다.

다음은 사전에 지정한 보조 분석이며 1차 정의를 대체하지 않는다.

- `large`와 `leader`를 따로 분석한 2×2 표
- 소형주를 하위 50%로 한정한 더 강한 소형주 가설
- 대형주 경계 상위 10%, 20%, 25% 민감도
- 시가총액 대장주가 아닌 252거래일 상대강도 1위의 “가격 대장주” 정의

보조 결과가 더 좋아 보여도 1차 정의를 교체하거나 그 결과만 결론에 사용하지 않는다.

## 7. 가설과 1차 추정량

두 진입 채널만 비교하기 위해 1차 실험에서는 **청산 채널을 모두 20일로 고정**한다.

- `R20,A`: 그룹 A에 20/20을 적용한 비용 차감 후, 동일 위험 기준 일별 수익률
- `R55,A`: 그룹 A에 55/20을 적용한 비용 차감 후, 동일 위험 기준 일별 수익률
- `R20,B`, `R55,B`: 그룹 B의 동등한 수익률
- `ΔA = mean(R20,A - R55,A)`
- `ΔB = mean(R20,B - R55,B)`
- **1차 상호작용 추정량 `I = ΔA - ΔB`**

가설 방향은 `ΔA > 0`, `ΔB < 0`, 따라서 `I > 0`이다. 서로 다른 두 부분집합의 p값을 따로 보고 “차이가 있다”고 결론 내리지 말고, 반드시 `I` 자체의 신뢰구간을 검정한다. 이는 관찰적 예측 관계이며 인과관계라고 표현하지 않는다.

## 8. 반드시 비교할 전략

동일한 종목군, 자본, 비용, 위험 한도와 거래 순서를 사용해 다음 네 전략을 비교한다.

| ID | 그룹 A | 그룹 B | 목적 |
|---|---:|---:|---|
| `ALL_20` | 20/20 | 20/20 | 전 종목 단기 채널 기준선 |
| `ALL_55` | 55/20 | 55/20 | 현재 장기 진입 기준선 |
| `ADAPTIVE` | 20/20 | 55/20 | 사용자 가설 |
| `REVERSE` | 55/20 | 20/20 | 방향성 위약 대조군 |

2차 실험에서만 고전 조합 `20/10` 대 `55/20`과 `A=20/10, B=55/20`을 비교한다. 20일 진입과 10일 청산을 동시에 바꾼 결과를 “20일 진입 효과”로 해석하지 않는다. 원래 Turtle System 1의 직전 승리 거래 건너뛰기와 55일 failsafe 같은 추가 규칙도 별도 실험으로만 허용하며 조용히 끼워 넣지 않는다.

## 9. 신호·체결·달력 규칙

1. 종목별 실제 거래일 바를 사용한다. 휴장일 carry-forward나 여러 자산의 합집합 달력 행 수를 20일·55일로 세지 않는다.
2. 진입 채널은 현재 바를 제외한다.
   - `H_N(t) = max(high[t-N], ..., high[t-1])`
   - 신호는 `close[t] > H_N(t)`일 때만 발생한다. `>=`는 사용하지 않는다.
3. 20일과 55일의 공정한 비교를 위해 양쪽 모두 종목별 55개 선행 바와 ATR 준비가 끝난 공통 시작일부터 측정한다. 20일 전략의 자연 시작 결과는 민감도로만 표시한다.
4. 1차 체결은 `t`일 종가 확인 후 해당 종목의 다음 실제 거래일 `t+1` 시가다. 당일 채널 가격에 이상적으로 체결된 것으로 가정하지 않는다.
5. 진입 다음 날 갭은 실제 시가와 비용 모형에 반영한다. 다음 바가 거래정지로 없으면 첫 거래 재개일로 이월한다.
6. 20일 청산은 현재 바를 제외한 이전 20일 저가 채널을 종가가 하향 돌파하면 다음 거래일 시가에 체결한다.
7. 보호 손절은 모든 전략에 동일하게 적용한다. ATR20의 정확한 계산식과 초기화법을 설정에 기록한다. 기존 순수 함수를 재사용하려면 의미를 단위 테스트로 확인한다.
8. 보호 손절이 활성화된 날 시가가 손절가보다 불리하면 시가, 장중 저가만 손절가를 통과하면 손절가에 슬리피지를 더한 가격으로 체결한다. 일봉에서 같은 날 여러 주문 순서가 불명확하면 가장 보수적인 손익 순서를 적용하고 기록한다.
9. 신호 가격에는 split-adjusted OHLC를 사용하고 현금배당은 별도 현금흐름으로 반영한다. 미래 현금배당까지 반영한 총수익 조정가격을 신호에 조용히 사용하지 않는다.
10. 데이터 종료 시 열린 포지션은 마지막 공통 거래 가능 종가에 비용을 반영해 강제 청산한 결과를 1차로 사용한다. 미실현 평가와 미청산 제외 결과를 모두 민감도로 표시한다.
11. 모든 반복, 정렬, 동률 처리와 난수 시드는 결정론적으로 만든다. 같은 입력·설정 해시는 바이트 단위로 같은 요약 결과를 만들어야 한다.

## 10. 포트폴리오와 위험 규칙

1차 정책 판단은 실제로 자본을 공유하는 포트폴리오 결과로 한다. 거래별 독립 1R 결과는 신호 자체 진단용 보조 분석이다.

- 1차: 진입당 계좌자산의 0.5% 위험, 총 위험 한도 12%, 단일 종목 평가액 한도 25%, 최대 1유닛, 피라미딩 없음
- 2차 앱 유사 민감도: 최대 2유닛과 현재 코드의 나머지 규칙. 1차 결과와 섞지 않는다.
- 모든 비교 전략은 같은 초기자본과 위험 한도를 사용한다.
- 같은 날 신호가 자본 한도를 넘으면 배열 순서로 선착순 체결하지 않는다. 가용 위험을 후보들에게 비례 배분하고 정수 주식 수로 내림한 뒤 잔여 현금은 보유한다.
- 미국과 한국은 우선 각 시장의 현지 통화 원장으로 독립 검증한다. 역사적 FX를 갖춘 통합 KRW 원장은 보조 정책 분석으로 제공한다.
- 시장별 결과를 합칠 때 한 시장의 우수 결과가 다른 시장의 반대 결과를 숨기지 않도록 개별 결과를 항상 함께 제시한다.

## 11. 거래비용·유동성·수용 가능 규모

- 매수와 매도 양쪽에 수수료, 스프레드, 슬리피지와 시장충격을 반영한다.
- 한국의 매도세 등 날짜에 따라 바뀐 비용은 해당 시점에 유효했던 공식 세율을 적용하고 출처와 시행일을 남긴다. 오늘의 세율을 과거 전체에 소급하지 않는다.
- 호가 데이터가 없으면 가격, 60일 중앙 거래대금, 변동성을 이용한 보수적 구간별 비용식을 결과 확인 전에 동결한다.
- 기본 비용, 비용 없음, 기본의 2배를 모두 계산하되 의사결정은 기본 비용과 2배 스트레스를 통과해야 한다.
- 주문금액이 이전 60거래일 중앙 거래대금의 5%를 넘는 체결은 1차 수용 가능 규모 분석에서 제한 또는 미체결 처리한다. 정확한 처리 방식을 사전등록한다.
- 전략별 회전율, 평균 스프레드 추정치, 미체결 신호 수, ADV 대비 주문비율과 AUM별 수용 가능 규모를 보고한다.

## 12. 통계 검정과 강건성

### 12.1 1차 검정

- 잠금 표본의 비용 차감 일별 차이로 `I`를 계산한다.
- 종목과 시장의 동시 충격을 보존하도록 60거래일 블록 부트스트랩을 사용한다.
- 최소 10,000회, 고정 시드로 95% 신뢰구간과 양측 p값을 계산한다.
- 미국, 한국, 두 시장 50:50 합산을 모두 보고한다.
- 1차 가설은 하나뿐이다. 보조 검정은 Holm 방식으로 다중비교 보정한다.

### 12.2 반드시 보고할 지표

- 비용 차감 CAGR, 연환산 변동성, Sharpe, Sortino, MDD, Calmar/MAR
- 거래 수, 승률, 평균·중앙값 R, Profit Factor, 평균 보유일, 노출도, 회전율
- 연도별·시장별·업종별·규모 분위별 성과
- `ΔA`, `ΔB`, `I`의 점추정·95% CI·p값
- `ADAPTIVE`가 `ALL_20`, `ALL_55`, `REVERSE` 각각에 대해 보인 paired 차이
- 총수익뿐 아니라 최대 낙폭과 하방 위험의 차이

### 12.3 사전 지정 강건성·위약 검정

- 진입 채널 주변값: 15/20/25일 및 45/55/65일. 우수값을 새 기본값으로 선택하지 않는다.
- 대형주 경계: 상위 10%, 20%, 25%.
- 공통 55바 시작 대 20일 전략 자연 시작.
- 강제 청산 대 미실현 평가 대 미청산 제외.
- 기본 비용 대 2배 비용.
- 기본 `t+1` 시가 체결 대 `t+2` 시가 체결 지연 스트레스.
- 종가 확인 후 익일 시가인 1차 체결 대 장중 고가 돌파·보수적 stop 주문 근사. 장중 근사는 보조로만 표시한다.
- 최대 1유닛 대 앱 유사 2유닛.
- 단일 최대 기여 종목, 업종, 연도를 하나씩 제외한 leave-one-out.
- 시장·월별 A 그룹 비중을 보존한 1,000회 무작위 라벨 순열. 실제 분류가 임의 분할보다 특별한지 확인한다.
- 종목별 `20일 채널 이점`을 종속변수로 하고 `large`, `leader`, 로그 시가총액, 거래대금, 변동성, turnover, 시장·연도 고정효과를 포함한 보조 회귀. 표준오차는 종목과 달력월로 이중 군집화한다.

민감도 중 일부만 가설에 유리하게 골라 본문 결론으로 올리지 않는다. 1차 결과의 부호가 공통/자연 시작, 종료 처리 또는 합리적인 비용 가정에서 뒤집히면 `INCONCLUSIVE`로 판정한다.

## 13. 판정 규칙 — 결과를 보기 전에 고정

최종 판정은 다음 셋 중 하나다.

- `SUPPORTED_FOR_PAPER_PILOT`
- `NOT_SUPPORTED`
- `INCONCLUSIVE`

`SUPPORTED_FOR_PAPER_PILOT`은 다음 조건을 **모두** 충족할 때만 허용한다.

1. 잠금 표본 50:50 합산에서 `I`의 95% CI 하한이 0보다 크다.
2. 잠금 표본 점추정이 `ΔA > 0`, `ΔB < 0`이다.
3. 미국과 한국의 `I` 점추정이 모두 양수다. 한 시장만 통과하면 글로벌 규칙을 채택하지 않고 시장 한정 후보로만 기록한다.
4. `ADAPTIVE`가 잠금 표본에서 `ALL_20`과 `ALL_55` 각각보다 비용 차감 CAGR이 최소 1.0%p, Sharpe가 최소 0.10 높다.
5. `ADAPTIVE`의 MDD가 두 전역 기준선 중 더 나은 값보다 2.0%p 넘게 악화되지 않는다.
6. 비용 2배에서도 `I > 0`이고 `ADAPTIVE`가 두 전역 기준선보다 열등하지 않다.
7. A와 B가 시장별 최소 30종목, 잠금 표본 최소 100회 청산 거래를 충족한다.
8. 최대 기여 종목·업종·연도를 각각 제거해도 `I`의 부호가 양수로 유지된다.
9. `ADAPTIVE`가 `REVERSE`보다 우수하며 무작위 라벨 정책의 95백분위보다 높다.
10. 룩어헤드, 생존편향, 기업행위, 중복 바, 휴장일 압축 또는 통화 오류가 데이터 QA에서 발견되지 않는다.

가설 방향과 반대이고 신뢰할 수 있는 충분한 데이터가 있으면 `NOT_SUPPORTED`다. 데이터 부족, 오염, 낮은 검정력, 결과의 설정 민감성 또는 위 조건 사이의 모순 때문에 판단할 수 없으면 `INCONCLUSIVE`다. 통과하더라도 곧바로 앱 기본값을 바꾸지 말고 paper/shadow pilot만 제안한다.

## 15. 필수 테스트

최소한 다음 사례를 자동 테스트한다.

- 현재 바가 Donchian 고가/저가 계산에 포함되지 않음
- 정확히 20/55개의 종목별 거래일을 사용하고 합집합 달력 행을 세지 않음
- 55바가 없는 종목은 공통 시작 전에 신호를 내지 않음
- 종가 확인 후 다음 실제 거래일 시가 체결
- 휴장·거래정지·갭 손절 처리
- 분할 전후 신호와 손익 연속성
- 상장폐지와 합병 대가 반영
- 월말 정보가 다음 달부터 적용되고 진입 그룹이 청산까지 고정됨
- `A = large OR leader`, `B = NOT large AND NOT leader`의 완전 분할
- 동률, 동일일 다중 신호와 정수 주식 반올림의 결정성
- 비용 0/기본/2배 및 매수·매도 양방향 비용
- 열린 포지션 종료 처리 세 방식
- 고정 입력·설정·시드에서 결과 해시 재현
- 작은 수작업 데이터셋에서 `ΔA`, `ΔB`, `I`와 paired 차이의 정답 대조

## 16. 검증 명령

새 전용 명령을 `package.json`에 추가하되 기존 명령을 깨지 않는다.

```text
npm run test:conditional-channel
npm run backtest:conditional-channel -- --phase=data-audit
npm run backtest:conditional-channel -- --phase=prelock
npm run backtest:conditional-channel -- --phase=lockbox
npx tsc --noEmit
npm run test:turtle
npm run test:actionqueue
npm run build
```

외부 데이터가 없어 lockbox를 실행할 수 없더라도, 데이터 감사·순수 로직 단위 테스트·타입 검사·기존 회귀 테스트는 가능한 범위까지 수행한다. 실패한 명령은 숨기지 말고 원인, 영향과 재현 명령을 최종 보고에 남긴다.
`data-audit`가 하드 게이트를 통과하지 못하면 `prelock`과 `lockbox` 명령은 성과 추정치를 생성하지 않고 명확한 비정상 종료 코드와 `DATA_GAP` 보고서를 반환해야 한다.

## 17. 최종 보고서 형식

보고서 첫 화면에 다음을 순서대로 둔다.

1. **판정:** `SUPPORTED_FOR_PAPER_PILOT / NOT_SUPPORTED / INCONCLUSIVE`
2. **한 문장 답:** 대형주·대장주에는 20일, 기타에는 55일이 더 낫다고 말할 수 있는지
3. **증거 등급:** `EXPLORATORY / PREREGISTERED REPLICATION`, 데이터 as-of, 설정 해시
4. **핵심 표:** `ΔA`, `ΔB`, `I`, 95% CI, `ADAPTIVE` 대 세 기준선
5. **채택 게이트:** 10개 조건의 PASS/FAIL/NA와 근거
6. **가장 중요한 한계와 다음 행동**

그 뒤에 데이터 계보와 QA, 종목군 정의, 실행 규칙, 전체 성과, 통계 검정, 시장·연도·업종·유동성 분해, 비용·수용 가능 규모, 강건성, 거래 집중도, 제외 목록, 재현 방법을 배치한다. 모든 표에 거래 수와 표본 기간을 함께 표시한다.

차트는 최소한 다음을 포함하되 표본이 부족하면 빈 그림 대신 부족 사유를 쓴다.

- 네 전략의 비용 차감 자산곡선과 drawdown
- A/B별 20일 대 55일 차이와 95% CI
- 미국/한국 및 연도별 `I`
- 시가총액 분위와 유동성 분위별 채널 이점
- 기본 비용과 2배 비용의 정책 비교

마지막에는 수정 파일, 실행한 명령과 결과, 미해결 데이터 문제, 재현 명령을 요약한다. 결과가 아무리 좋아도 이 작업에서 앱 기본값을 직접 변경하지 않는다.

## 실행 중 절대 금지

- 현재 보유종목만으로 “전체 시장에서 검증됐다”고 결론 내리기
- 오늘의 시가총액·업종·상장 종목 목록을 과거에 소급하기
- 현재 바를 돌파 채널에 포함하기
- 여러 시장의 합집합 달력을 종목별 거래일처럼 세기
- 당일 돌파 가격에 무조건 이상 체결하기
- 상장폐지·거래정지·기업행위를 조용히 버리기
- 테스트 구간 또는 잠금 표본을 보고 임계값·기간·제외 규칙을 변경하기
- `20/10`과 `55/20` 차이를 순수한 진입 채널 효과라고 부르기
- 별도 부분집합 p값만으로 상호작용을 주장하기
- 유리한 시장·연도·비용·룩백만 골라 보고하기
- 데이터가 없는데 합성값 또는 현재값으로 채워 확증 결과를 만들기
- 기존 보고서 숫자를 새 실행 결과인 것처럼 복사하기

이 작업의 최우선 목표는 긍정적인 결론이 아니라 **재현 가능하고 반증 가능한 결론**이다.

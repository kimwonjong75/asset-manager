## 1. 질문

기존 투더문(비터틀) 보유분에 터틀 규칙을 **오늘부터 부착**하는 것이, **그대로 관찰만** 하는 것보다 나은가?

목적은 수익 극대화가 아니라 **손실 방치 제거(실행 문제 해결)** 이므로 1차 판정축은 하방 통제(MDD)다.

### P2 상세 (부록 A 결정 4)
- 편입 체결가 = **D+1 실제 시가** (실매매 아님 — 기존 보유분의 터틀 기준가. 편입 시 매매비용 없음)
- N = **D까지의 ATR20**
- 손절가 = 편입 체결가 − 2N
- 청산 = 직전 20거래일 최저가 (당일 제외)
- 기존 과다 보유분 = `LEGACY_EXCESS`로 기록하되 **1유닛 규격 충족으로 간주하지 않음**
- 신규매수·불타기: 잔여예산 0이므로 **미실행**. 불타기 가격조건 충족 시 `SIGNAL_BLOCKED_BY_BUDGET` 기록('신호 없음'과 구분)

## 3. 기준선 규칙 (동결)

55일 진입 / 20일 청산 / 2N 손절 / +0.5N 불타기 / 최대 2유닛 / ATR20 / 돈치안 당일 제외.

**체결 규약:** 종가[t] 판정 → **다음 실제 거래일 시가[t+1] 체결 + 비용**.
`min(open,stopPrice)`·`min(open,exitLow)`·`max(open,trigger)` **사용 금지**. 갭 위험은 익일 실제 시가에 이미 반영된 것으로 처리한다. 장중 high/low 터치·상시 스톱주문은 **민감도 전용**(1차 미실행).
판정 우선순위: 손절 > 20일 청산 > 불타기 (엔진 `buildTurtleActions` 순서와 동일).

## 4. 리스크·예산 회계 (승인된 6개 수정 유지)

- `riskPerUnitPct = 0.5`는 **사이징** 값: 수량 = 예산 × 0.5% ÷ N.
- **2N 손절 시 실손실 = 예산의 1%** (= riskPerUnitPct × stopMultipleN). 0.5%를 손절위험 상한으로 쓰지 않는다. 엔진 `positionRiskAtStop`/`computeTotalOpenRisk`와 정의 일치.
- 불타기 후 위험은 1% 가산이 아니라 **상향된 공통 손절가로 전체 포지션 재계산** + 잔여예산 재검사.
- 불타기 트리거 N = **현재 market N** (`evaluatePyramid`와 동일). `turtlePositionView.nAtLastFill` 표시값 재사용 금지(D-1).
- 25% 종목 상한은 **전체 포지션 기준**. 총위험 한도 12%.
- 위성예산 = **총자산의 10%**, 보유비중에 맞춰 상향 금지. 일반 SATELLITE 보유분도 **deployed에 포함**.
- Phase 0 실측: deployed 17.65% > 예산 10% → **잔여예산 0** (본인분만 14.18%로도 초과 → 결론 불변).
- 기존 보유 종목 **중복 신규진입 금지**.

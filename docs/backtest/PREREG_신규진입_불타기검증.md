## 4. 규칙 (동결 — config.json 이 정본)

진입 55일(당일 봉 제외) / 청산 20일(당일 봉 제외) / N = D일까지 ATR20 / 손절 = 진입 또는 마지막 불타기 체결가 − 2N / 유닛 = 예산 0.5% ÷ N / 총 오픈위험 ≤ 12% / 종목 전체 시가 ≤ 25% / 재진입은 완전 청산 후 새 돌파에서만.

**수량 granularity(결과 확인 전 동결):** 암호화폐(BTC-USD·ETH-USD)는 소수(1e-8 내림), 그 외는 정수 내림. 앱 엔진 관례(`allowFractional = isCrypto`)와 동일. *정수만 허용하면 BTC/ETH 는 유닛 사이징 결과가 1주 미만이라 항상 0주가 되어 거래가 전무해진다(침묵 실패).*

**체결:** 신호는 D일 종가 → **해당 종목의 다음 실제 거래일 시가** 체결. `max(open,돌파선)`·`min(open,손절선)` 보정 **금지**. 갭은 실제 손익에 그대로 반영. 휴장일 신호·체결 없음. 평가용 종가만 이월(시가·고가·저가 이월 금지). 대기주문은 체결 전 현금·포지션·위험 불변. 같은 날 매도 → 매수 순. 우선순위 손절 → 청산 → 불타기. 매도 대기 포지션에 불타기 주문 없음. 종목별 중복 포지션·중복 대기주문 금지.

**같은 날 다중 매수:** 선착순 금지 — 일괄 계산 후 **공통 비율 λ** 하향(수량 내림). 후보를 티커 오름차순 정렬해 입력 순서 무관을 보장한다.

**시험하지 않는 것:** 20/10 채널, 다른 ATR·손절값, 최대 3·4유닛, 새 부트스트랩·추가 통계모형·다른 기간분할.

## 8. 고정 판정 기준 (결과 확인 전 동결)

`1R = 실제 첫 진입 수량 × 2N × 체결일 환율` (KRW). 완료거래 손익(비용 차감) ÷ 1R.

### T1 신규진입 신호 사용 조건 (전부 충족)
완료거래 ≥ 30건 · 기본비용 평균 R > 0 · 2배비용 평균 R > 0 · 기본비용 CAGR > 0 · 3구간 중 ≥2에서 평균 R > 0 · 회계·안전 불변식 위반 0.
완료거래 < 30건 → **INCONCLUSIVE** 종료(설정 변경·종목 추가 금지).

### T2 불타기 채택 조건 (T1 통과가 선결 + 전부 충족)
실제 불타기 체결 ≥ 20건 · 기본비용 ΔCAGR ≥ +1.0%p · 2배비용 ΔCAGR > 0 · T2 Calmar > T1 · T2 MDD 악화 ≤ 2.0%p · 3구간 중 ≥2에서 ΔCAGR > 0 · 최대 증분기여 종목 제거 후 ΔCAGR > 0 · 한 종목이 증분손익 50% 초과 안 함 · 불변식 위반 0.
불타기 체결 < 20건 → **INCONCLUSIVE** 종료, 불타기 기본 OFF.

### 최종 판정
- T1 통과 + T2 통과 → `신규진입 신호 사용 / 불타기 켬`
- T1 통과 + T2 실패·표본부족 → `신규진입 신호 사용 / 불타기 끔`
- T1 실패·표본부족 → `신규진입 추천 보류 / 불타기 끔`

**"신호 사용" = 관찰용 매수 후보로 사용 가능하다는 뜻이며 실제 주문 자동승인이 아니다.**

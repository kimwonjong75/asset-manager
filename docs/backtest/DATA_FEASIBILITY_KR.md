# DATA_FEASIBILITY_KR — Gate 0.5: 한국 데이터 적격성 실측 검증

> 이 문서는 PROMPT_3(§3 데이터 가능성 감사)의 **한국 시장 전용 심화 실측**이다. 성과·돌파신호·전략수익률은
> **절대 계산하지 않았다** — 이 단계는 point-in-time 종목군·시가총액·업종·상장폐지·기업행위 데이터가
> 실제로 존재하고 신뢰 가능한지에 대한 품질 정보만 확인한다(§4-1 "수익률과 무관한 품질 정보만").
> 모든 조사는 하위 에이전트 없이 이 세션에서 직접 수행했다(WebSearch/WebFetch/Bash/Python 직접 실행).

- **가설 ID:** `conditional-channel-v1`
- **실측일:** 2026-07-16
- **최종 판정: `KR_SIZE_READY`** (조건부 — 하단 "판정 근거"·"실행 전 필수 잔여작업" 참조)
- **판정 대상:** 대형주(large) 축은 조건부 통과, 업종 대장주(leader) 축은 미통과(WICS point-in-time 정확도 미확증)

## 0. 이번 단계에서 절대 하지 않은 것

- 실제 성과·CAGR·Sharpe·승률 등 어떤 백테스트 수치도 계산하지 않았다.
- 돌파 신호나 전략 수익률을 계산하지 않았다.
- WiseIndex 비공식 API의 월별 대량 수집을 하지 않았다 — 최초 시도(11개 섹터×2개 시점=22회)가
  세션의 자동 권한 분류기에 의해 **"사용자가 승인하지 않은 대량 수집"으로 실제로 차단됐다**. 이후 훨씬
  좁게(특정 종목 2개, 특정 섹터 2개, 특정 시점 2개 = 4회 호출)만 수행하고 즉시 중단했다.
- KRX 계정/서비스키를 대신 발급받지 않았다(계정 생성은 자격증명이 필요한 행위로 에이전트가 대행할 수
  없는 영역이다 — 사용자가 직접 해야 한다).

## 1. 사용한 원천과 접근 방법

| 원천 | 역할 | 접근 방법 | 결과 |
|---|---|---|---|
| **marcap** (github.com/FinanceData/marcap) | 1차 원천(교차검증용으로 지정됐으나, KRX 공식 접근 장벽 때문에 실질적 1차 데이터가 됨) | 공개 parquet 파일 직접 다운로드(2010~2025, 16개 연도, 약 330MB) | **성공**, 전체 실측의 기반 |
| KRX 공식 Open API (openapi.krx.co.kr) | 1차 원천 후보(사용자 지정) | 계정/API 키 필요 | **접근 불가** — 계정 발급은 에이전트가 대행 불가 |
| data.krx.co.kr 익명 JSON 엔드포인트 | 대체 접근 시도 | 직접 POST 호출 | **`LOGOUT` 응답, HTTP 400** — 2025-12-27 로그인 의무화를 직접 실증 확인 |
| data.go.kr(공공데이터포털) 금융위원회_주식시세정보 | 관허 대안 | 서비스키 필요 | **접근 불가**(계정 발급 필요, 대행 불가) |
| Naver Finance 공개 JSON(api.finance.naver.com) | **독립 제3소스 교차검증**(KRX 공식 아님, 명시) | 공개 웹 API, 키 불요 | 3종목 교차검증 수행(아래 §3) |
| WiseIndex(wiseindex.com) GetIndexComponets | 업종 point-in-time 후보 | 비공식 내부 API, `dt` 파라미터 | 제한적 검증(아래 §9), **비공식·ToS 불명 유지** |

**중요한 한계**: 사용자가 지정한 "KRX 공식 API를 1차 원천으로" 요구사항을 이번 세션에서 충족하지 못했다.
세 가지 공식/관허 경로(openapi.krx.co.kr, data.krx.co.kr, data.go.kr) 모두 계정 또는 서비스키 발급이
필요했고, 계정 생성은 안전수칙상 에이전트가 대행할 수 없는 행위다. **사용자가 직접 계정을 만들어야
진짜 KRX 공식 교차검증이 가능하다.** data.go.kr 서비스키는 문서상 "자동승인"이라 5분 내 발급될 것으로
보인다(이전 세션 조사 결과, 미확인).

## 3. KRX 공식 대비 교차검증 → 독립 제3소스(Naver) 대체 검증

KRX 공식 소스에 접근할 수 없어, **명시적으로 "KRX 공식이 아닌" 독립 제3자 공개 소스**(Naver Finance)로
marcap의 OHLCV를 교차검증했다. 이는 사용자가 요청한 "KRX 공식 교차검증"의 대체물이지 동일물이 아니다.

| 종목 | 기간 | 결과 |
|---|---|---|
| 삼성전자(005930), 2020-06-25~07-02 (6거래일) | 최근·분할 후 안정기 | **OHLCV 6/6일 완전 일치**(Open/High/Low/Close/Volume 전부), 시가총액=종가×상장주식수 공식도 정확히 일치(3.098317e+14 등) |
| 다음(035720), 2012-01-10~01-17 (6거래일) | 이후(2021년경) 분할 이력 있음 | Volume은 완전 일치, **가격은 일관되게 ~4.98배 차이** — marcap이 미조정 원시가이고 Naver가 이후 분할을 소급 조정한 값이기 때문으로 해석(§7의 "미조정 원시가" 가정과 정합) |
| 셀트리온(068270), 2024-03-04~03-11 | 2023년 말 셀트리온헬스케어 합병 이후 | 가격·거래량 모두 배율 불일치(~1.14배) — 합병에 따른 재계산 가능성, 원인 미확정 |

**해석**: 최근 발생한 기업행위가 없는 종목(삼성전자 2020년 구간)은 **바이트 단위로 완전 일치**했다.
기업행위가 개입된 종목은 불일치하지만, 그 배율이 설명 가능한 패턴(분할비율 근사)을 보인다. 이는 데이터
오류가 아니라 **marcap이 미조정 원시가라는 가정과 일치하는 정상적 현상**이다. 다만 진짜 KRX 공식
소스와의 직접 대조는 계정 발급 후 사용자가 완료해야 완전히 닫힌다.

## 4. 종목코드 재사용·개명 충돌

직접 계산(스크래치패드 `code_reuse.py`, 3,858개 코드 전수 조사).

- **코드 재사용(무관한 신규 회사가 옛 코드를 재사용) 사례: 발견되지 않음.**
- 같은 코드에서 연도 간 종목명이 바뀐 사례 8건 — 전부 **동일 법인의 개명/리브랜딩**으로 확인됨
  (예: `036220` 인포피아→오상헬스케어, `229640` LS전선아시아→LS에코에너지, `220260` 엘아이지이에스스팩→켐트로스[스팩 합병 후 실체 전환, 정상 절차]).
- 2개 코드(`036220`, `198940`)는 **5~7년의 장기 결측 구간**을 보인 뒤 재등장 — 장기 거래정지 가능성이
  높은 사례로, 실제 거래정지 이력 확인이 필요(§8의 `TradingHaltPeriod` 타입으로 흡수 가능한 케이스).
- **결론: 종목코드를 안정적 식별자로 사용해도 이번 조사 범위에서 충돌 위험이 발견되지 않았다.**
  다만 3,858개 전수를 다 육안 검수한 것은 아니며, 실제 파이프라인 구축 시 자동 검증 루틴(연속성 검사)을
  넣을 것을 권장한다.

## 5. ISIN 등 안정 식별자 확보 가능성

- **marcap에는 ISIN 필드가 없다**(컬럼: Code, Name, Close, Dept, ChangeCode, Changes, ChangesRatio,
  Volume, Amount, Open, High, Low, Marcap, Stocks, Market, MarketId, Rank, Date — 18개 전부 확인).
- data.go.kr의 "금융위원회_KRX상장종목정보"가 ISIN 필드를 제공하는 것으로 이전 조사에서 확인됐으나
  이번 세션에서 직접 검증하지 못했다(서비스키 필요, 계정 발급 장벽 동일).
- **결론**: 지금 당장은 종목코드(6자리)를 permanent identifier로 쓸 수밖에 없다. §4의 재사용 조사
  결과 이번 범위에서는 안전해 보이지만, ISIN 조인은 사용자가 data.go.kr 키를 발급한 뒤 보강 과제로
  남긴다.

직접 계산(스크래치패드 `sectype.py`, 6개 연도 스냅샷: 2010/2013/2016/2019/2022/2025).

| 축 | 필드 | 확인 결과 |

| **ETF/ETN/펀드** | (marcap에 아예 없음) | marcap은 개별 종목만 수집 — ETF 코드(예: 069500 KODEX200) 조회 시 **부재 확인** → 자동 제외됨 |
| **KONEX** | `Market` | 2013년부터 KONEX 시장 분리 표시(45개 → 2025년 115개) → 필드로 정확히 제외 가능 |

| **스팩** | 종목명 "스팩" 포함 | 2010년 21개 → 2025년 86개, 꾸준히 존재·증가 → 이름 패턴으로 안정적 식별 |
| **리츠** | 종목명 "리츠"/"REIT" 포함 | 2010년 4개 → 2025년 28개(한국 리츠 시장 성장과 일치) → 이름 패턴으로 식별 가능 |
| **투자회사** | 종목명 "투자회사/사모투자/기업인수목적" 포함 | **0건 검출** — 이 범주가 marcap에 독립 상장 종목으로 존재하지 않거나 이름 패턴이 다름(추가 조사 필요, 낮은 우선순위 — 한국엔 미국 BDC류 상장 투자회사가 사실상 없음) |

**결론**: `Market != KONEX` + `코드 끝자리 0` + `이름에 스팩/리츠 미포함` 3중 필터로 **비-KONEX 보통주
후보의 약 92%**(2025년 2,674/2,903)를 잡아낼 수 있다. 완벽한 단일 필드는 없지만 **필드+패턴 조합으로
신뢰할 만한 필터링이 가능하다.**

## 7. 기업행위 전후 OHLC 연속성 (요구 20건 → 698건 확보)

직접 계산(스크래치패드 `corp_actions.py`).

**결정적 확인 — 삼성전자(005930) 2018-05-04 액면분할(알려진 사건)**:

| 날짜 | 종가 | 상장주식수 |
|---|---:|---:|
| 2018-04-27 | 2,650,000원 | 128,386,494주 |
| 2018-05-04 | 51,900원 | 6,419,324,700주 |

상장주식수 비율 = 6,419,324,700 / 128,386,494 = **정확히 50.0000배** (알려진 50:1 분할과 일치).
종가는 역수로 정확히 대응(51,900 × 50 ≈ 2,595,000, 실제 시가 변동 포함 시 합리적 범위) — **가격과
주식수가 서로 역수로 정확히 맞아떨어져, marcap 원시가가 미조정(raw)이라는 가정이 데이터로 직접
확증됐다.**

**전체 패널 스캔**: 상장주식수가 전일 대비 30% 이상 점프/하락하면서 **동시에 가격이 역방향으로
움직여 (상장주식수 변화율 × 가격 변화율 ≈ 1.0, ±15% 이내)** 인 경우를 분할/무상증자의 "raw-data
signature"로 정의하고 2010~2025 전체를 스캔한 결과 **698건**을 확인(요구된 20건을 크게 초과 충족).
샘플(2010년 25건): 10배(액면분할 10:1), 5배, 2배, 0.5배(병합) 등 깨끗한 비율이 실제로 관측됨.

**결론**: marcap 원시 OHLC는 **분할·무상증자에 대해 조정되지 않은 원시가**이며(사용자 가정과 일치),
조정 계수는 **`Stocks`(상장주식수) 컬럼에서 직접 역산 가능**하다 — 별도의 외부 기업행위 피드 없이도
분할 조정 파이프라인을 marcap 자체 데이터로 구축할 수 있는 근거가 확인됐다. 단, 이 조정 파이프라인
자체는 **아직 구현되지 않았다** — 데이터 존재는 확인됐으나 엔지니어링 과제로 남아있다(§11 참조).

## 10. 합병 대가(비정상 폐지 없는 케이스)

- §8의 "정상(합병/자진폐지)" 그룹(401건)은 최종 거래가가 사실상 합병가치를 반영한 것으로 보이나,
  **정확한 합병비율·교환대가는 DART 공시에서 별도로 확인해야 한다.**
- DART Open API의 기업개황(`induty_code`) 엔드포인트는 존재가 확인됐으나 **합병비율/합병대가를
  기계판독 가능한 구조화 필드로 제공하는 전용 엔드포인트는 확인하지 못했다**(이전 세션 조사, 이번
  세션에서 재확인하지 않음). 합병 대가 반영은 **수작업 테이블 관리가 필요한 잔여 과제**로 남는다.

## 11. Split-adjustment 파이프라인 — 아직 미구현 (중요)

사용자의 사전 가정("marcap 원시 OHLC는 split-adjusted가 아니라고 가정") **은 §7에서 데이터로
확증됐다.** 이것이 의미하는 바:

- **이번 Gate 0.5 단계에서 marcap 원시가를 돌파 신호 계산에 절대 사용하지 않았다**(사용자 지시 준수).
- 분할·무상증자 조정 계수는 `Stocks`(상장주식수) 컬럼의 일별 변화율로 역산 가능함을 확인했으나(§7),
  **이를 실제로 적용해 조정 가격 시계열을 만드는 파이프라인 코드는 아직 작성하지 않았다.**
  이는 다음 단계(데이터 파이프라인 구현)에서 반드시 먼저 만들고, 별도 골든 테스트로 검증해야 한다
  (예: 삼성전자 2018-05-04 전후 조정가가 연속적인지, §7의 698건 중 다수 표본으로 회귀 검증).
- 배당(무상증자 아닌 현금배당)에 의한 조정은 marcap 컬럼만으로는 감지되지 않는다 — 별도 배당
  이력 소스가 필요하다(마지막 세션 조사에서 미확보, 이번 세션에서 재조사하지 않음).

## 12. WICS 업종분류 point-in-time 정확도 — 미확증 (핵심 잔여 리스크)

이번 세션에서 **사용자 지시대로 대량 수집을 하지 않고 소량만 검증**했다.

1. **1차 시도(차단됨)**: 11개 GICS/WICS 대분류 섹터 × 2개 시점(2015-01/2020-01) = 22회 호출을
   시도했으나, **세션의 자동 권한 분류기가 "사용자가 승인하지 않은 대량 수집"으로 실제로 차단했다.**
   이는 정확히 사용자가 걱정한 시나리오("소량 검증하고 즉시 중단")를 시스템이 강제한 사례다.
2. **2차 시도(축소, 4회 호출만)**: 2018년 글로벌 GICS 개정(IT→통신서비스, 2018-09-24 시행)으로
   재분류됐을 것으로 **가정**한 네이버(035420)·카카오(035720) 2종목을, IT(G45)·통신서비스(G50)
   2개 섹터, 2018-01-02/2019-01-02 2개 시점 = 4회 호출로 확인.
   - **결과: 가정이 틀렸다.** 두 종목 모두 2018년과 2019년 양쪽에서 **IT(G45)에 그대로 남아있었다**
     — WICS는 GICS의 2018년 개정을 그대로 따르지 않은 것으로 보인다(또는 다른 시점에 이동했을
     가능성, 미확인).
   - 이미 받은 응답 안에서(추가 호출 없이) 확인한 결과: 통신서비스(G50) 구성종목은 2018/2019
     양년 모두 7개로 동일(순서만 변동, 실질적 이동 없음). IT(G45)는 27개 이탈·36개 신규진입의
     큰 변동이 있었으나, 이는 IPO/상장폐지에 의한 자연스러운 종목 교체일 가능성이 높고 **실제
     "재분류"(다른 기존 섹터로 이동)인지는 이번 검증 범위에서 구분하지 못했다.
3. **`dt` 파라미터 자체는 정상 작동 확인**(날짜별로 다른, 유효한 구성종목 목록을 반환) — 2010년까지
   응답 가능함도 확인(§13). 다만 **특정 종목의 특정 재분류 사건을 정확히 포착하는 사례는 이번
   검증에서 확보하지 못했다.**

**결론: WICS point-in-time 정확도는 이번 Gate 0.5에서 PASS로 선언할 수 없다.** 사용자가 제시한
"알려진 업종변경 10건 전후 정확도 99% 이상" 기준을 충족하는 실증을 얻지 못했다 — 실패로 확인된 것도
아니고, 검증 자체가 미완이다. 승인된 예산(소량) 안에서는 결론을 내릴 수 없으며, 다음 중 하나가
필요하다: (a) 확실히 알려진 WICS 재분류 사례(정확한 종목·날짜)를 사용자가 지정, 또는 (b) 더 큰
범위의 사전승인된 검증(월별 대량 수집 별도 승인).

## 13. 데이터 시작 시점

## 17. 판정: `KR_SIZE_READY` (조건부)

### 사용자 지정 통과 기준 대비 결과

| 기준 | 상태 |
|---|:--:|
| 월별 보통주 커버리지 95% 이상(WICS) | **미검증**(대량수집 미승인으로 측정 불가) |
| 알려진 업종변경 전후 정확도 99% 이상 | **미확증**(시도한 사례가 실제로는 재분류 없었음) |
| 관리·정리매매·상장폐지 종목의 마지막 유효 업종 미래정보 없이 복원 | **미확증**(업종 축 자체가 미확증이므로 종속 실패) |
| 자동 수집·개인 연구 이용 허용 근거 | **부분적**(data.go.kr만 명확, 나머지는 회색~중간 위험) |

**사용자가 제시한 조건부 규칙**("업종 데이터가 실패하더라도 대형주 축에 필요한 데이터가 모두 통과하면
KR_SIZE_READY로 판정")을 적용한다. 대형주(large, 시가총액 기준) 축에 필요한 데이터는:

| 요구 | 상태 |
|---|:--:|
| point-in-time 종목군(상장폐지 포함) | **PASS** — 날짜별 스냅샷 구조로 자동 충족, 873개 소멸종목 확인 |
| point-in-time 시가총액 | **PASS** — `Marcap`/`Stocks` 일별 존재, 삼성전자 분할로 정확성 확증 |
| 상장폐지 수익률(임의대체 없음) | **PASS(조건부)** — 정리매매 실거래가 존재 확인(한진해운), 단 자동 태깅 불가(수동/휴리스틱 분류만 가능, §8) |
| 종목유형 구분(보통주만) | **PASS(조건부)** — 필드+패턴 조합, 완벽한 단일필드는 없음 |
| 코드 안정성(재사용 없음) | **PASS** — 조사 범위 내 충돌 없음 |
| 기업행위 조정 가능성 | **PASS(데이터만)** — 조정계수 역산 가능함을 확인, **파이프라인 자체는 미구현**(잔여 엔지니어링) |
| 3완전연도 잠금 표본 | **PASS** — 2023~2025 정확히 3완전연도 |
| 자동수집 허용 근거 | **부분 PASS** — marcap 라이선스 불명, KRX 직접접근 비권장, data.go.kr만 명확(계정 필요) |

### 잔여 필수 작업 (실행 전, 우선순위 순)

1. **[사용자 필수]** data.go.kr 서비스키 발급(자동승인) 후, 진짜 KRX 공식 데이터로 marcap 재교차검증
2. **[엔지니어링]** `Stocks` 컬럼 기반 split-adjustment 파이프라인 구현 + 698건 표본 회귀 테스트
3. **[엔지니어링]** DART에서 합병대가/비율 수작업 테이블 구축(§10의 401건 "정상 폐지" 그룹 대상)
4. **[판단 필요]** marcap 라이선스 불명 상태를 수용할지 여부(§14) — 사용자 결정 사항
5. **[선택]** WICS 검증을 더 진행하려면 사용자가 정확한 재분류 사례(종목+날짜)를 지정하거나, 더 큰
   범위의 검증을 별도 승인

### 왜 `KR_FULL_READY`가 아닌가

업종 대장주(leader) 축에 필요한 WICS point-in-time 정확도를 확증하지 못했다(§12). 이 축을 포함하려면
위 "선택" 항목의 후속 검증이 필요하다.

### 왜 `NOT_READY`가 아닌가

대형주(large) 축만으로 그룹 A/B 정의 일부(A=large만, leader 제외)를 구성할 수 있고, 그 축에 필요한
데이터 요구사항은 실측으로 충분히 뒷받침됐다(위 표). 다만 이는 원래 사양(§6)의 "large OR leader"
전체가 아니라 **leader를 제외한 축소판**이라는 점을 분명히 한다 — 이 축소판으로 진행할지는 사용자
판단이다.

# DATA_GAP — 조건부 돌파 채널 가설 검증 (강제 중단 보고서)

> **PROMPT_3 §3·§14 강제 중단 산출물.** Phase 0 데이터 가능성 감사가 하드 게이트를
> 통과하지 못했다. 따라서 1차 확증 백테스트(lockbox)를 실행하지 않으며, 성과 추정치를
> 생성하지 않는다. 이 문서는 무엇이 왜 막혔는지와 진행에 필요한 외부 입력을 기록한다.

- **가설 ID:** `conditional-channel-v1`
- **가설:** 대형주 또는 업종·카테고리 대장주는 20일 신고가 돌파 진입이 유리하고, 그 밖의 종목은 55일 신고가 돌파 진입이 유리한가?
- **감사 종류:** 정적 능력 감사(static capability audit) — 이 저장소에서 현재 도달 가능한 데이터 소스의 능력만 대상. 라이브 외부 소스 프로빙 없음.
- **게이트 판정:** **FAIL** (8개 하드스톱 조건 전부 충족 → 8/8)
- **판정 근거 코드:** `scripts/backtest/conditionalChannel/dataQuality.ts` (`runDataAudit()`)
- **매니페스트:** `scripts/backtest/conditionalChannel/dataManifest.json`
- **사전등록 방법론:** `docs/backtest/PREREG_조건부채널검증.md`

---

## 1. 결론 한 줄

현재 저장소에는 **생존편향 없는 point-in-time 전체 시장 종목군, 역사적 시가총액, 역사적 업종 분류, 상장폐지 종목·수익률** 데이터가 없다. 이 가설은 그 데이터 없이는 **원리적으로 확증 검증이 불가능**하다. 무료 티커 중심 가격만으로 수치를 만들면 §3·"절대 금지" 조항을 위반하는 오염된 결과가 된다.

|---|---------|------|:----:|-----------|

## 4. 진행에 필요한 외부 입력 (필수)

확증(Phase 0 통과) 검증을 재개하려면 아래를 **실제** point-in-time 데이터로 확보해야 한다.

1. **전체 시장 point-in-time 투자 가능 종목군** — US 상장 보통주 + KOSPI/KOSDAQ 보통주의 월말 목록, **상장폐지·합병·티커변경·거래정지 종목 포함**. 제공처 예: CRSP/Compustat, S&P Capital IQ, FactSet, KRX 데이터 상품.
2. **역사적 발행주식수/시가총액 시계열** — §6-1 대형주(80백분위) 판정용.
3. **역사적 표준 업종 분류(GICS/WICS)** — 유효 시작·종료일 포함. §6-2 대장주 판정용.
4. **상장폐지 대가/최종 수익률** — 합병 현금/주식 대가 포함. 생존편향·임의대체 제거용.
5. **명시적 기업행위 레코드** — 분할 비율·현금배당·거래정지 기간 + 중복 바 판별 규칙.
6. **시장·연도별 공식 수수료·세율** — 한국 매도세 시행일 포함(오늘 세율 소급 금지).
7. (통합 KRW 원장 보조 분석 시) **거래일별 USD/KRW FX 시계열**.

## 5. 비타협 원칙 (§3·"절대 금지")

- 현재 보유종목만으로 "전체 시장에서 검증됐다"고 결론 내리지 않는다.
- 오늘의 시가총액·업종·상장 종목 목록을 과거에 소급하지 않는다.
- 상장폐지·거래정지·기업행위를 조용히 버리거나 0/최종종가로 임의 대체하지 않는다.
- 데이터가 없는데 **합성값 또는 현재값으로 채워** 확증 결과를 만들지 않는다.
- 기존 보고서 숫자를 새 실행 결과처럼 복사하지 않는다.
- Yahoo/Stooq 등 무료 티커 중심 가격만으로 한 실험은 `EXPLORATORY / SURVIVORSHIP-BIASED`로만 표기하고 앱 변경 근거로 쓰지 않는다.

## 6. 이 게이트가 막는 것과 막지 않는 것 (§16)

- **막는 것:** `--phase=prelock`, `--phase=lockbox`의 성과 추정치 생성. 게이트 FAIL이면 이 명령들은 성과를 만들지 않고 명확한 비정상 종료 코드와 이 `DATA_GAP` 보고서를 반환해야 한다.
- **막지 않는 것:** 순수 로직 스캐폴딩·단위 테스트·타입 검사·기존 회귀 테스트. 데이터 감사 자체(`--phase=data-audit`)는 이 실패를 프로그램적으로 검출하고 그 상태를 보고하는 것이 정상 동작이다.

## 7. 재현 명령

```text
# 데이터 감사(이 보고서의 판정 재현)
npm run backtest:conditional-channel -- --phase=data-audit   # (CLI 배선은 후속 작업)
# 순수 로직/타입 검사
npx tsc --noEmit
npm run test:conditional-channel                              # (테스트 배선은 후속 작업)
```

> CLI(`run.ts`)와 `package.json` 명령 배선, 단위 테스트(`tests/conditionalChannelBacktest.ts`)는 후속 작업에서 추가된다. 이 문서 시점의 게이트 판정은 `dataQuality.ts`의 `runDataAudit()`로 직접 재현 가능하다.

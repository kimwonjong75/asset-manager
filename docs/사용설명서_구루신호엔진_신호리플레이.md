# 구루 신호 엔진 & 신호 리플레이 사용 설명서

> 주식 차트의 이동평균선, RSI 정도만 아는 초보자를 위한 안내서입니다.  
> 목적은 하나입니다. **상승/하락 구간에서 신호가 제대로 떴는지 확인하고, 틀린 신호를 기록해 더 나은 규칙 후보를 찾는 것**입니다.

---

## 0. 먼저 결론부터

이 기능은 자동 매매 버튼이 아닙니다. 앱 화면에도 반복해서 나옵니다.

> 지식 규칙 기반 참고 신호이며 투자자문이 아닙니다.

쉽게 말하면:

| 기능 | 한마디로 | 쓰는 곳 |
|---|---|---|
| 구루 신호 엔진 | 지금 내 종목에 신호가 떴는지 보는 화면 | 대시보드 |
| 신호 리플레이 | 과거 차트에서 그 신호가 맞았는지 채점하는 화면 | 신호 리플레이 |
| 규칙 샌드박스 | 조건값을 바꿔 보며 더 나은 기준을 찾는 연습장 | 신호 리플레이 |

가장 중요한 흐름은 이렇습니다.

```text
종목 선택
→ 상승/하락 구간 찾기
→ 그 자리에 매수/매도 신호가 떴는지 확인
→ 적절함 / 너무 빠름 / 너무 늦음 / 잘못된 신호 / 놓친 매수 / 놓친 매도로 기록
→ 샌드박스에서 조건값을 바꿔 보기
→ 새 신호가 좋아졌는지 연구용/검증용 사례로 다시 확인
```

샌드박스에서 바꾼 값은 **실제 라이브 신호에 반영되지 않습니다.** 현재 기능은 “좋아 보이는 규칙 후보를 찾고 기록하는 단계”입니다. 실제 신호 공식에 영구 반영하는 것은 별도 개발/반영 단계가 필요합니다.

## 1. 20분 빠른 실습

처음이라면 아래 순서만 따라 해도 핵심을 이해할 수 있습니다.

### 1단계: 종목 하나 고르기

`신호 리플레이` 탭으로 들어갑니다. 검색창에 종목명이나 티커를 입력합니다.

예:

```text
AAPL
TSLA
삼성전자
SLV
```

보유 종목이나 관심 종목이 있으면 검색창 아래 칩을 눌러도 됩니다.

### 2단계: 기간과 모드 고르기

처음에는 이렇게 시작하세요.

| 항목 | 추천 |
|---|---|
| 기간 | 1년 |
| 모드 | 복기(미래 공개) |

`복기(미래 공개)`는 이후 차트까지 보면서 “그때 신호가 맞았는지” 채점하기 좋습니다.  
익숙해지면 `리플레이(미래 숨김)`으로 바꿔 실전처럼 연습합니다.

### 3단계: 큰 상승 구간 찾기

차트에서 “여기서부터 꽤 올랐다” 싶은 구간을 찾습니다.

그 상승 초입에 초록 매수 마커가 있었는지 봅니다.

| 보이는 모습 | 판정 |
|---|---|
| 상승 시작 전후에 매수 신호가 뜸 | 적절함 |
| 상승이 한참 진행된 뒤 매수 신호가 뜸 | 너무 늦음 |
| 매수 신호가 떴는데 곧바로 무너짐 | 잘못된 신호 |
| 명확히 좋은 자리인데 아무 신호도 없음 | 놓친 매수 |

### 4단계: 큰 하락 구간 찾기

이번에는 “여기서부터 많이 빠졌다” 싶은 구간을 찾습니다.

하락 전후에 매도 경고 마커가 있었는지 봅니다.

| 보이는 모습 | 판정 |
|---|---|
| 하락 전 고점 근처에 매도 경고가 뜸 | 적절함 |
| 너무 일찍 매도 경고가 떠서 이후에도 계속 오름 | 너무 빠름 |
| 이미 많이 빠진 뒤 매도 경고가 뜸 | 너무 늦음 |
| 위험해 보였는데 매도 경고가 전혀 없음 | 놓친 매도 |

### 5단계: 내 판정 저장하기

날짜를 클릭한 뒤 `내 판정` 패널에서 버튼을 누릅니다.

```text
적절함 / 너무 빠름 / 너무 늦음 / 잘못된 신호 / 놓친 매수 / 놓친 매도
```

메모에는 한 줄만 적어도 충분합니다.

예:

```text
상승 초입인데 거래량 조건 때문에 신호 없음
고점 이후 10일 지나서 매도 경고가 떠서 늦음
매수 신호 후 바로 MA20 아래로 이탈
```

이 기록이 쌓이면 “앱이 자주 놓치는 패턴”이 보입니다.

## 2. 화면 지도

### 대시보드: 구루 신호 엔진

대시보드의 `구루 신호 엔진` 카드는 현재 보유 종목과 관심 종목을 검사합니다.

화면에 이런 안내가 나옵니다.

```text
지식 규칙 기반 관찰 후보 · 매수 추천이 아닙니다
활성 규칙 N개 평가
왜 신호가 안 뜨나요?
```

여기서는 “지금 어떤 종목을 살펴봐야 하나?”를 봅니다.

### 신호 리플레이

`신호 리플레이` 탭은 과거 데이터에 현재 구루 규칙을 다시 적용합니다.

화면 안내의 핵심은 이 문장입니다.

```text
현재 활성 구루 규칙을 과거 가격에 다시 적용한 결과입니다.
그날 실제 발송된 알림이 아닙니다.
시점을 옮기면 그날까지 데이터만으로 신호를 재계산합니다(미래 미반영).
```

즉, 리플레이는 “그때 실제 팝업이 떴는지”를 복원하는 화면이 아닙니다.  
**현재 규칙을 과거의 각 날짜에 대입해 보는 검증 도구**입니다.

## 3. 차트와 기호 읽는 법

### 차트 마커

| 마커 | 뜻 |
|---|---|
| 초록 ▲ `매수 N` | 구루 매수/관찰 신호 |
| 빨강 ▼ `매도 N` | 구루 매도 경고 |
| 노란 ◆ | 지금 선택한 기준 날짜 |

`N`은 그날 동시에 걸린 구루 규칙 수입니다. 예를 들어 `매수 2`면 매수 쪽 구루 규칙 2개가 같은 날 발화했다는 뜻입니다.

중요한 점:

> 차트 마커와 이전/다음 신호 이동은 구루 신호 기준입니다.

RSI, 골든크로스, 데드크로스 같은 일반 가격 알림은 차트 마커가 아니라 `가격기반 알림 진단`에서 따로 봅니다.

### 이동평균선과 RSI

차트에는 아래 선이 같이 표시됩니다.

| 표시 | 초보자용 해석 |
|---|---|
| MA5 | 아주 짧은 흐름 |
| MA20 | 단기 추세 중심선, 눌림목 확인에 자주 사용 |
| MA60 | 중기 흐름 |
| MA120/MA150 | 긴 추세 확인 |
| RSI(14) | 과열/침체를 보는 보조 지표 |

RSI는 보통 70 이상을 과열, 30 이하를 침체로 봅니다. 다만 강한 상승 종목은 RSI가 오래 높게 유지될 수 있고, 약한 하락 종목은 RSI가 오래 낮게 유지될 수 있습니다. RSI 하나만 보고 판단하지 마세요.

### 진단 기호

구루 신호 엔진과 리플레이 진단에는 같은 기호가 나옵니다.

| 기호 | 뜻 | 행동 |
|---|---|---|
| ✓ | 조건 충족 | 신호가 뜬 이유 |
| ✗ | 조건 미충족 | 신호가 안 뜬 이유 |
| - | 판정 불가 | 데이터 없음 또는 앱 미지원 |

각 줄은 보통 이렇게 읽습니다.

```text
조건 이름 / 실제값 / 기준값
```

예:

```text
거래량 비율 1.10 (기준 1.20)
```

실제값 1.10이 기준 1.20보다 낮으니 거래량 조건은 미충족입니다.

## 4. 현재 구루 신호 규칙

현재 코드 기준으로 자동 발화 가능한 주요 구루 규칙은 아래 4개입니다.

| 규칙 | 앱이 찾으려는 자리 | 초보자용 설명 |
|---|---|---|
| 클라이맥스 톱 과열 매도 경고 | 너무 가파른 상승 뒤 과열 | “많이 뜨거우니 줄일지 보자” |
| MA20 눌림목 관찰 | 상승 추세 중 MA20 근처 조정 | “강한 종목이 잠깐 쉬는 자리일 수 있다” |
| 신고가 근접 돌파 관찰 | 52주 고점 근처 + 거래량 동반 | “강한 종목이 다시 뚫을지 보자” |
| MA20 재돌파 관찰 | MA20 아래로 갔다가 다시 회복 | “무너진 듯했지만 다시 살아나는지 보자” |

`RS 90+ 리더 종목 필터` 같은 규칙은 현재 초안/미지원 지표가 있어 자동 발화 대상이 아닐 수 있습니다. 진단에서 `미지원`이나 `비활성`으로 보이면 전략이 틀렸다는 뜻이 아니라, 앱이 아직 그 지표를 계산하지 못하거나 자동 신호로 승인되지 않았다는 뜻입니다.

## 5. 대시보드에서 구루 신호 엔진 쓰기

대시보드의 `구루 신호 엔진`은 오늘의 신호판입니다.

### 신호가 있을 때

종목은 액션별로 묶입니다.

| 라벨 | 뜻 |
|---|---|
| 관찰 후보 | 아직 매수 확정이 아니라 지켜볼 후보 |
| 진입 검토 | 매수 검토 단계에 가까운 후보 |
| 매도 경고 | 비중 축소나 매도 점검이 필요한 위험 신호 |
| 리스크 | 포지션 크기나 손실 한도 관련 |
| 시장 국면 | 시장 전체 분위기 필터 |
| 복기 | 지난 흐름 점검 |

종목 옆의 `왜 떴나`를 누르면 어떤 조건 때문에 신호가 떴는지 보입니다.

### 신호가 없을 때

`왜 신호가 안 뜨나요?`를 누릅니다.

진단 종목을 선택하면 아래처럼 구분해서 보여줍니다.

| 상태 | 뜻 |
|---|---|
| 충족 - 신호 발화 | 조건을 만족해 신호가 떴음 |
| 미충족 | 데이터는 있는데 기준을 못 넘음 |
| 일부 데이터 누락 | 일부 지표가 없어 완전 판단 불가 |
| 데이터 부족 | 필요한 가격/거래량 데이터가 부족 |
| 미지원 | 앱이 아직 계산하지 못하는 지표 |
| 비활성 | 초안, 미승인, 만료 등으로 꺼진 규칙 |

초보자는 여기서 딱 하나만 보면 됩니다.

> 신호가 안 뜬 이유가 “조건 미충족”인지, “데이터 문제”인지, “앱 미지원”인지 구분하세요.

보정할 수 있는 것은 주로 `조건 미충족`입니다. `데이터 부족`과 `미지원`은 규칙값을 만져도 해결되지 않을 수 있습니다.

## 6. 신호 리플레이 기본 조작

### 종목 선택

검색창에 종목명이나 티커를 입력합니다. 보유/관심 종목은 칩으로 바로 선택할 수 있습니다.

종목을 고르면 앱이 2015년 이후 과거 시세를 불러와 리플레이 타임라인을 만듭니다. 데이터가 부족하면 “데이터 부족” 또는 “시세를 불러오지 못했습니다”가 보일 수 있습니다.

### 모드 선택

| 모드 | 언제 쓰나 |
|---|---|
| 리플레이(미래 숨김) | 그날 이후를 가리고 실전처럼 판단할 때 |
| 복기(미래 공개) | 이후 실제 상승/하락을 보며 채점할 때 |

신호 계산은 두 모드 모두 그 날짜까지의 데이터만 사용합니다. `복기(미래 공개)`는 차트만 미래를 보여주는 채점 모드입니다.

### 날짜 이동

| 버튼 | 기능 |
|---|---|
| 이전 신호 | 직전 구루 신호일로 이동 |
| -1일 | 하루 전 거래일 |
| +1일 | 하루 뒤 거래일 |
| 다음 신호 | 다음 구루 신호일로 이동 |
| 최신 | 가장 최근 날짜 |

차트 위 날짜를 클릭해도 해당 날짜로 이동합니다.

## 7. 상승 구간 검증법

상승 구간 검증은 “살 만한 자리에서 매수 신호가 있었나?”를 보는 과정입니다.

### 좋은 매수 신호의 모습

대체로 이런 위치가 좋습니다.

```text
바닥 확인
→ MA20 회복 또는 MA20 근처 눌림
→ 거래량이 붙거나 신고가 근처로 접근
→ 이후 상승
```

리플레이에서는 이 구간에 초록 ▲가 있었는지 봅니다.

### 판단 기준

| 상황 | 내 판정 |
|---|---|
| 상승 초입 또는 건강한 눌림목에서 매수 신호 | 적절함 |
| 너무 바닥 초기에 떠서 오래 흔들림 | 너무 빠름 |
| 이미 많이 오른 뒤 고점 근처에서 매수 신호 | 너무 늦음 |
| 매수 신호 뒤 바로 하락 전환 | 잘못된 신호 |
| 명확한 상승 초입인데 신호 없음 | 놓친 매수 |

### 무엇을 메모할까

메모는 길 필요 없습니다.

예:

```text
MA20 근처였지만 거래량 기준 때문에 미발화
52주 고점 기준이 너무 빡빡해서 놓침
디스트리뷰션 조건은 통과했는데 고점과 너무 멀었음
```

이 메모가 나중에 샌드박스 조정의 근거가 됩니다.

## 8. 하락 구간 검증법

하락 구간 검증은 “피해야 할 자리에서 매도 경고가 있었나?”를 보는 과정입니다.

### 좋은 매도 경고의 모습

대체로 이런 위치가 좋습니다.

```text
급등 또는 과열
→ 거래량 급증
→ 상승 탄력 둔화
→ 이후 하락
```

리플레이에서는 이 구간에 빨강 ▼가 있었는지 봅니다.

### 판단 기준

| 상황 | 내 판정 |
|---|---|
| 하락 전 고점 근처에서 매도 경고 | 적절함 |
| 너무 일찍 떠서 이후에도 계속 상승 | 너무 빠름 |
| 이미 많이 하락한 뒤 매도 경고 | 너무 늦음 |
| 매도 경고가 떴지만 이후 강하게 상승 지속 | 잘못된 신호 |
| 위험 구간인데 경고 없음 | 놓친 매도 |

### 주의

매도 경고는 “내일 폭락” 예언이 아닙니다. 과열은 며칠 또는 몇 주 더 이어질 수 있습니다. 그래서 “매도 경고 후 바로 떨어졌나”만 보지 말고, **위험을 미리 줄이는 데 도움이 됐는지**를 봐야 합니다.

## 9. 샌드박스로 보정하기

`규칙 샌드박스`는 조건값을 바꿔 보며 신호가 어떻게 달라지는지 확인하는 곳입니다.

화면 설명:

```text
임계값·on/off 조정(이 화면만, 라이브 미반영)
값을 바꾸면 차트 마커와 신호가 즉시 다시 계산됩니다.
실제 구루 신호·알림·저장된 규칙은 바뀌지 않습니다.
```

### 언제 보정하나

보정은 아무 때나 하는 게 아닙니다. 먼저 판정을 남긴 뒤 합니다.

| 발견한 문제 | 샌드박스에서 의심할 것 |
|---|---|
| 놓친 매수가 많음 | 조건이 너무 엄격한가 |
| 잘못된 매수가 많음 | 조건이 너무 느슨한가 |
| 매도 경고가 너무 늦음 | 과열 기준이 너무 높은가 |
| 매도 경고가 너무 자주 뜸 | 과열 기준이 너무 낮은가 |

### 보정 예시

예시는 방향을 이해하기 위한 것입니다. 무조건 이렇게 바꾸라는 뜻은 아닙니다.

| 문제 | 조정 후보 |
|---|---|
| 신고가 근처 돌파를 자주 놓침 | `52주 고점 대비 거리` 기준을 조금 완화 |
| 거래량이 살짝 부족해서 좋은 자리를 놓침 | `거래량 비율` 기준을 조금 낮춰 보기 |
| 약한 반등에도 매수 신호가 자주 뜸 | `52주 고점 근접` 조건을 더 엄격하게 |
| 하락 전 과열 경고가 늦음 | `클라이맥스 플래그` 기준을 낮춰 보기 |
| 과열 경고가 너무 남발됨 | `클라이맥스 플래그` 기준을 높이거나 조건을 다시 켜기 |

### diff 읽기

값을 바꾸면 상단에 변화가 뜹니다.

| 표시 | 뜻 |
|---|---|
| 기준 대비 신호일 변화 없음 | 바꿨지만 결과가 같음 |
| + N일 추가 | 새로 생긴 신호 날짜 |
| - N일 사라짐 | 없어져 버린 신호 날짜 |

좋은 보정은 보통 이런 모습입니다.

```text
원하던 상승 초입에는 매수 신호가 추가됨
엉뚱한 고점 매수 신호는 사라짐
검증용 사례에서는 성과가 무너지지 않음
```

나쁜 보정은 이런 모습입니다.

```text
내가 방금 본 한 종목에는 딱 맞는데 다른 종목에서는 신호가 난사됨
검증용 사례에서 신호일이 크게 흔들림
조건을 너무 많이 꺼서 사실상 아무 기준도 안 남음
```

### 되돌리기

| 버튼 | 기능 |
|---|---|
| 한 줄의 되돌리기 | 해당 조건만 원래대로 |
| 규칙 초기화 | 그 규칙 전체 원래대로 |
| 전체 초기화 | 모든 샌드박스 조정 제거 |

## 10. 내 판정과 놓친 신호 관리

`내 판정`은 이 기능의 핵심 데이터입니다. 신호를 고치는 재료가 바로 여기서 나옵니다.

### 판정 버튼 6개

| 버튼 | 언제 누르나 |
|---|---|
| 적절함 | 신호 위치가 좋음 |
| 너무 빠름 | 신호가 너무 일찍 뜸 |
| 너무 늦음 | 이미 움직인 뒤 뜸 |
| 잘못된 신호 | 방향이 틀림 |
| 놓친 매수 | 좋은 매수 자리인데 신호 없음 |
| 놓친 매도 | 위험한 매도 자리인데 신호 없음 |

`놓친 매수`와 `놓친 매도`는 신호가 없는 날에도 기록할 수 있습니다.

### 놓친 매수/매도 모아보기

화면 아래 `놓친 매수/매도 모아보기`는 내가 기록한 누락 사례를 종목별로 모아줍니다.

이 패널은 이렇게 쓰면 좋습니다.

```text
어떤 종목에서 놓친 매수가 반복되는가?
어떤 조건 때문에 자주 놓치는가?
놓친 매도는 과열 기준 문제인가, 데이터 문제인가?
```

여기에 많이 쌓이는 종목과 조건이 개선 1순위입니다.

## 11. 검증 사례: 과적합을 막는 안전장치

샌드박스 보정은 쉽게 과적합으로 흐릅니다.

과적합이란:

```text
내가 방금 본 과거 차트에는 기가 막히게 맞지만
새로운 차트에서는 전혀 안 맞는 상태
```

그래서 사례를 두 종류로 나눕니다.

| 종류 | 용도 | 규칙 |
|---|---|---|
| 연구용 | 조건값을 찾아보는 연습문제 | 여기서 보정 |
| 검증용(holdout) | 처음 보는 시험지 | 여기에 맞춰 고치지 않기 |

좋은 사용법:

```text
연구용 사례에서 조건 조정
→ 괜찮아 보이면 저장
→ 검증용 사례 재실행
→ 검증용에서도 악화되지 않는지 확인
```

화면에서도 검증용 사례가 흔들리면 과적합 경고를 띄웁니다.

```text
검증용(holdout) 사례입니다.
이 사례에 규칙을 맞추지 마세요(과적합).
```

## 12. 규칙별 성과 보기

`규칙별 성과` 표는 현재 종목/기간에서 신호가 얼마나 괜찮았는지 요약합니다.

| 항목 | 뜻 |
|---|---|
| 발화 | 신호가 뜬 횟수 |
| 적중률 | 매수는 20일 후 상승, 매도는 20일 후 하락이면 적중 |
| 평균 20일 | 신호 후 20거래일 평균 수익률 |
| 최대 상승/하락 | 신호 후 60거래일 안의 최대 움직임 |

주의:

> 괄호 안 표본 수가 적으면 적중률을 믿지 마세요.

예를 들어 신호가 2번 떠서 2번 맞으면 적중률은 100%지만, 표본이 너무 적어 우연일 수 있습니다. 여러 종목과 여러 기간에서 반복 확인해야 합니다.

## 13. 초보자용 추천 루틴

매주 30분만 한다면 아래처럼 하세요.

### 월 1회: 넓게 보기

```text
관심 종목 5개 선택
→ 기간 1년 또는 2년
→ 큰 상승/하락 구간만 체크
→ 놓친 매수/놓친 매도 위주로 기록
```

```text

```

```text
한 번에 조건 하나만 바꾼다.
좋아진 이유를 메모한다.
검증용 사례에는 맞추지 않는다.
표본이 적으면 결론을 미룬다.
```

## 14. 자주 헷갈리는 질문

### Q. 차트에 매수/매도 마커가 하나도 없어요.

그 기간에 구루 신호가 없었던 것입니다. 정상일 수 있습니다.  
대시보드나 리플레이의 `왜 신호가 안 뜨나요?` 진단에서 조건 미충족인지, 데이터 부족인지, 미지원인지 확인하세요.

### Q. RSI나 골든크로스는 왜 차트 마커에 안 뜨나요?

차트 마커는 구루 신호 기준입니다. RSI, 골든크로스, 데드크로스 같은 가격기반 알림은 `가격기반 알림 진단`에서 봅니다.

### Q. 샌드박스에서 바꾼 값이 저장되지 않아요.

정상입니다. 샌드박스는 미리보기입니다. 실제 구루 신호, 알림, 저장된 규칙은 바뀌지 않습니다. 중요한 발견은 판정 메모와 검증 사례로 남기고, 필요하면 JSON 백업을 해두세요.

### Q. 좋은 보정값을 찾으면 바로 믿어도 되나요?

아니요. 연구용 사례에서 좋아진 뒤 검증용 사례에서도 나빠지지 않는지 확인해야 합니다. 한 종목, 한 구간에만 맞으면 과적합일 수 있습니다.

### Q. `신호 후 결과` 숫자는 신호 계산에 쓰이나요?

아니요. `신호 후 결과`는 미래를 본 채점용 숫자입니다. 신호 계산 자체는 그 날짜까지의 데이터만 사용합니다.

## 15. 마지막 주의

이 문서와 앱 신호는 투자 판단을 대신하지 않습니다.

꼭 기억할 것:

```text
신호는 후보를 보여줄 뿐이다.
좋은 신호도 틀릴 수 있다.
과거에 맞은 보정이 미래에도 맞는다는 보장은 없다.
샌드박스 보정은 실제 신호에 자동 반영되지 않는다.
데이터가 부족하거나 미지원이면 신호가 안 뜰 수 있다.
```

외부 일반 자료와도 교차 확인했습니다. RSI는 보통 14기간, 70/30 기준이 널리 쓰이지만 단독 사용보다 다른 지표와 함께 보는 것이 안전하다는 설명이 일반적입니다. 백테스트도 과거 성과를 보는 데 유용하지만, 과거에 맞춘 규칙이 미래에 그대로 통한다는 보장은 없고 과적합 위험이 있습니다.

참고:

- [Investopedia - RSI Indicator](https://www.investopedia.com/articles/active-trading/042114/overbought-or-oversold-use-relative-strength-index-find-out.asp)
- [Investopedia - Using Trading Indicators Effectively](https://www.investopedia.com/articles/trading/12/using-trading-indicators-effectively.asp)
- [Wikipedia - Backtesting](https://en.wikipedia.org/wiki/Backtesting)

최종 기준은 항상 현재 앱 화면입니다. 앱이 업데이트되면 이 문서의 일부 라벨이나 동작 설명도 바뀔 수 있습니다.
